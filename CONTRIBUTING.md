# 贡献指南

Universe Genesis Sandbox 当前正从阶段 0 文档进入阶段 1 实现。所有贡献都应遵守 [docs/phase-0.md](docs/phase-0.md) 与 [docs/milestones.md](docs/milestones.md) 中定义的阶段边界。

## 当前重点

阶段 1 要实现第一个“可复现宇宙卡片”最小可用版本：

1. 网页应用骨架。
2. `rulesetVersion` 与 seed 标准化。
3. 确定性 PRNG 与命名随机流。
4. 10 个初始宇宙模板。
5. `UniverseSummary`、`UniverseLaws` 与 `TimelineEvent`。
6. 宇宙指标与 8 至 12 条关键纪元事件。
7. 固定 seed 样本，用于复现性检查。

## 贡献规则

- 变更必须对齐当前阶段。
- 除非为阶段 1 解阻，否则不要引入阶段 2 及之后的系统。
- 保持确定性生成：同一 `seed + rulesetVersion + templateId` 必须得到同一结果。
- 模拟代码中不要使用不可控随机源。
- 法则、指标、事件和解释必须保持结构化，不要只写成叙述文本。
- 修改产品边界、数据契约或阶段门禁时，必须同步更新文档。
- 分享体验要保持轻量：用户可见文案优先简短，完整复现信息由分享码或链接参数承载。

## 合并请求检查清单

- 变更属于当前阶段，或是已接受的文档修正。
- 涉及生成逻辑时，已经检查固定 seed 行为。
- 新增生成字段有清晰的名称、标签和解释。
- 修改用户可见行为时，已同步 README 或文档目录。
- 没有混入无关重构。
