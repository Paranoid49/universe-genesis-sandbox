# 非功能需求

状态：步骤 1 已完成；当前发布快照仍是步骤 1 发布事实。步骤 2 的单步转换、连续运行、暂停、快照、恢复和存储预算已经在 `step-2-freeze.md` 中冻结，运行状态、逻辑时钟与可恢复随机流基础已经开始实现。

本文件是性能、兼容性、无障碍、可靠性和错误处理的当前事实来源。涉及固定模板、预制时间线、传统空间、奇迹、I1 分享和 A1 图书馆的要求只保护旧基线，退出时间以 [旧产品形状测试退役登记](legacy-test-retirement.md) 为准。

## 1. 性能与容量

- 旧基线普通测试环境中连续生成 50 个静态宇宙应在 2 秒内完成，覆盖率插桩环境允许 5 秒；步骤 2 用状态转换吞吐、暂停响应、快照和恢复预算替换。
- 兼容入口 `generateUniverse` 采用首次访问时重放、构建并缓存因果图的策略；产品入口 `generateCausalUniverse` 必须在返回宇宙前强制物化、校验并冻结因果图。
- 因果查询必须保留完整底层图。界面可以分批渲染，但筛选、直接关系和双向遍历必须能够访问全部节点。
- 当前因果界面首批展示 80 个结果、最多渲染 24 条完整路径并预览 80 个可达节点；发生截断时必须明确告知，并提供继续筛选或查看关系的文字路径。
- 观察与 Seed 对比的界面投影只能在玩家点击具体原因入口时调用 `appendCausalProjections`；普通页面渲染不得为全部坐标、强度、事件关联或左右对比结果预先复制完整因果图。
- 生产构建后的 JavaScript gzip 总体积不得超过 115 KiB，日常保护线为 110 KiB；调整依据见 [步骤 1 JavaScript 体积预算决策](decisions/bundle-budget-step-1.md)。
- 生产构建后的 CSS gzip 总体积不得超过 10 KiB，日常保护线为 8 KiB。
- 生产与 UI 测试采用 Preact React-compat，当前完整构建为 JavaScript gzip 87406 字节、CSS gzip 5828 字节；依据见 [步骤 1 用户界面运行时体积决策](decisions/ui-runtime-size-step-1.md)。

当前发布快照：模拟测试 104 项；模拟覆盖率 92.26%／83.44%／94.52%／95.72%；UI 测试 44 项；UI 覆盖率 89.96%／81.96%／92.3%／92.76%；独立性能测试 1 项；E2E 54 项通过、2 项跳过；JavaScript gzip 87406 字节；CSS gzip 5828 字节。
- 任意观察和可视化能力必须保留可操作的文字路径，不得要求先加载重型可视化运行时。
- 步骤 2 必须定义单步转换、连续运行、暂停响应、状态快照和恢复的自动化预算。
- 步骤 3 必须定义观察信号查询、证据历史和大对象集合分页预算。
- 步骤 4 必须定义分支创建、重放和差异计算预算。
- 步骤 7 必须定义长期分支数量、因果历史规模、IndexedDB、导入导出和迁移预算。

步骤 1 的独立非覆盖率性能门禁执行两次预热后，测量十二个多模板样本的完整生成、首次强制物化和双向查询，P50 与 P90 均须小于 1500 毫秒。覆盖率环境继续用代表性场景验证同一路径小于 3000 毫秒；仅依赖 50 个未访问惰性因果图的旧生成性能测试不能证明因果查询性能。

当前定向测试组成：步骤 1 契约测试 21 项；生产工厂守卫 2 项；架构测试 9 项；合计 32 项。

## 2. 兼容性

- 支持具备 ES2022、Web Crypto、URLSearchParams 和 Clipboard API 的现代桌面及移动浏览器。
- 页面最低适配宽度为 360 像素，当前响应式断点为 1180 像素与 720 像素。
- 因果查询在窄屏下不得产生页面级横向溢出，结果选择、方向切换和文字路径仍须可操作。
- 剪贴板 API 不可用时必须提供手动复制回退。
- 非法表单输入必须在页面内显示可访问错误，不得让同步生成异常逃逸到渲染层。
- 应用顶层必须提供未预期异常恢复界面。
- 模拟核心不得读取浏览器、网络、本地存储或系统时间状态。
- 当前生产与 UI 测试运行时为 Preact React-compat；React 18.3.1 与 `@types/react@18.3.31`、`@types/react-dom@18.3.7` 只提供源码和开发工具兼容契约，生产构建不得同时打入 React 与 Preact 两套运行时。

步骤 1 的兼容边界如下：

- 规则版本为 `ugs-ruleset@0.7.0`，短码为 `UGS070`。
- 因果契约版本为 `ugs-causality@13`，随机结果绑定版本为 `ugs-random-bindings@5`，显式输入清单版本为 `ugs-causal-inputs@1`，跨宇宙比较证据版本为 `ugs-law-comparison-evidence@1`，随机算法版本为 `fnv1a32-mulberry32@1`。
- 当前模拟核心内容哈希基线为 `b0f7669a66aabd662b67aebf8925ea923680c64fd1e190c4d312ec84b72fbebe`；本轮增强加权候选资格、全页面可见结果值和统计深层证据，不改变旧领域 JSON，因此规则版本仍为 `ugs-ruleset@0.7.0`。
- 观察与解释投影属于冻结的非生产视图，不得通过事实图公共认证、序列化或跨宇宙事实比较。
- `UniverseSummary` 继续作为旧页面兼容读取模型，并新增不可配置、非枚举、惰性缓存且运行时深度冻结的只读 `causalGraph`；旧 JSON、A1 和分享载荷不得隐式序列化整张因果图。
- UGS070 分享、I1 干预载荷、A1 图书馆和旧页面暂时保持可用。
- A1 仍只保存分享码和少量用户元数据，不把因果图冒充可继续运行的状态存档。
- 不支持的规则、因果、分享或存档版本必须明确拒绝、受控迁移或静态只读打开，不得按当前规则重建后声称等价。

UI 运行时必须在以下任一条件出现时重新评估：新步骤需要 React 专有能力、关键依赖无法在 Preact React-compat 下正确运行、真实浏览器出现无法局部关闭的兼容缺陷、旧界面拆除后 React 能重新满足体积保护线，或者总体积预算模型发生正式变更。每个产品步骤开始前必须复核。

## 3. 无障碍

- 目标为 WCAG 2.1 AA 的核心交互要求。
- 创世、导航、因果查询、观察、实验、干预、分支和存档控件必须具有可访问名称。
- 因果结果列表必须支持键盘进入、方向键连续选择和可见焦点。
- “为什么发生”和“产生了什么后果”必须使用正确的标签与面板语义，并支持键盘切换。
- 所有因果关系、规则和抽样证据必须有纯文字路径，不能只依赖颜色、图标、连线或悬停。
- 空原因、空后果、缺失节点、循环和路径截断必须提供可读说明。
- 自动化组件与真实浏览器测试不得出现 axe 可检测的严重无障碍违规。

## 4. 确定性与数据完整性

- 完备原因必须产生完全一致的结果和因果图。
- 随机算法版本、Seed 指纹、流 ID、完整命名空间、样本数量和一基抽样序号必须稳定；每次实际决策还必须记录操作类型、类型化范围／概率／候选／权重参数、可选领域作用域、候选集标识、受控候选快照、原始样本和选中值。
- 因果图构建和查询不得消费新的随机样本。
- 每个因果随机引用只能绑定一条实际决策，并同时通过流、命名空间、样本序号、候选集标识和选中值校验，不能引用一整段命名空间作为近似原因。
- 随机引用中的 `decisionId` 必须实际存在于其声明的 `streamId` 中，不能拼接其他随机流中数值相同的决策。
- 每个引用必须具有独立结果绑定，保存结果节点、结果主题、节点类型、类型化领域定位器、领域值指纹、作用域和字段／集合／负事实语义；生产认证必须针对受控重放结果解析定位器并核对领域语义，不能使用待认证图自报的期望值。
- 随机调用转录必须通过不授予认证能力的逐字段比较器核对六类操作、范围边界、概率、候选顺序、候选内容、标签和权重。
- 认证写入能力不得通过命名导出、默认导出、导出类、静态或实例方法、导出赋值及其 helper、别名、属性、回调和解构绕行公开，普通结构夹具、克隆、同步修改副本和重新冻结对象均不能自行获得生产认证。
- 公共成功认证结果必须冻结且不可污染；普通失败结果每次独立创建，调用方修改不得改变后续校验和序列化结论。
- 相同显示值不得成为合并因果节点或历史的依据。
- 因果节点、边、规则引用、双向邻接、根列表和随机引用必须在进入产品查询路径前通过闭包校验。
- 模板配置和创世初态必须作为两个独立 `initial_state` 根进入闭包，不能与 Seed、模板选择或规则版本等外部输入混同。
- 所有 `input` 根必须与规范化生成清单一一对应；清单按 Seed、规则版本、创世模板和有序干预排列，并与随机追踪共享同一生成身份。额外、缺失、重排、类型错配、值错配或跨宇宙随机追踪必须拒绝。
- 因果闭环只有在精确声明成员节点、内部边、授权高阶公理、全部一致性约束且存在外部合法根路径时才允许；未经授权或授权不完整的循环必须拒绝。
- 基础因果图和点击后追加的投影图在返回调用方前都必须完成闭包断言并深度冻结。
- 主要旧页面的可见领域结果必须能够直接进入唯一因果节点，并保存返回原页面继续浏览的上下文；定位失败不得静默忽略。
- 解释、名称与观察文案只能由事实和因果数据派生，不得反向写回模拟事实。
- 纯界面浏览不进入宇宙历史；未来若宇宙规则允许观察产生作用，该观察必须成为显式因果输入。
- 二维观察的摘要、坐标／尺寸／亮度、各强度值和事件关联必须各自引用实际计算操作数；强度值与来源主题由同一份投影度量定义生成，区域外文明或神话不得成为局部强度原因。Seed 对比必须建立绑定左右生成身份、左右来源节点、两侧评分、差值与固定顺序裁决的组合证据；左右随机历史仍保留在各自因果图中，不得合并或伪造跨图边。

## 5. 错误处理

- 旧生成入口继续校验 Seed、规则版本、模板和干预。
- 新因果校验错误必须使用稳定问题码并提供中文说明。
- 重复节点／边、悬空引用、非法根、输入清单错配、孤立节点、缺失或未知规则、邻接不一致、无根路径、非法循环和无效随机引用不得静默忽略。
- 因果查询目标不存在时应显示明确空状态，不得自动替换为另一个结果并隐瞒错误。
- 无效领域目标不得静默回退，必须抛出带错误码和字段路径的 `UniverseInputError`。
- 因果投影引用不存在、重复或类型不匹配的领域主题时必须明确拒绝，不能自动改指向其他对象。
- 损坏或不支持的分享载荷必须明确拒绝、受控迁移或静态只读打开。
- 存储写入失败时不得先提交内存成功状态；导入并发时不得覆盖导入开始后的本地变更。

## 6. 发布与进程管理

- 发布前必须执行 `npm run check:release`。
- 本地手动启动开发服务使用 `npm run dev`，停止方式是在同一可见终端按 `Ctrl + C`。
- 本地手动启动预览服务使用 `npm run preview -- --port 4173`，测试结束后必须按 `Ctrl + C` 停止。
- 任务结束前必须确认没有 Vite、Playwright 或其他测试服务器残留，并清理本次生成且无需交付的覆盖率、构建和测试报告目录。
