# 非功能需求

本文件是性能、兼容性、无障碍、可靠性和错误处理的唯一事实来源，阶段规格不再重复定义这些长期要求。

## 1. 性能预算

- 普通测试环境中连续生成 50 个宇宙应在 2 秒内完成，覆盖率插桩环境允许放宽到 5 秒。
- 法则对比只在用户进入法则页面时计算，不得在每次普通创世时额外生成对比宇宙。
- 生产构建后的 JavaScript 总压缩体积不得超过 110 KiB，CSS 总压缩体积不得超过 10 KiB。
- 阶段 7 新增可视化后，文字浏览路径仍须保持可操作，不得要求先加载重型可视化运行时。

## 2. 兼容性

- 支持具备 ES2022、Web Crypto、URLSearchParams 和 Clipboard API 的现代桌面及移动浏览器。
- 页面最低适配宽度为 360 像素，现有响应式断点为 1180 像素与 720 像素。
- 剪贴板 API 不可用时必须提供手动复制回退，不得导致页面崩溃。
- 非法表单输入必须在页面内显示可访问错误，不得让同步生成异常逃逸到渲染层。
- 应用顶层必须提供未预期异常恢复界面。
- 模拟核心不得读取浏览器、网络、本地存储或系统时间状态。

## 3. 无障碍

- 目标为 WCAG 2.1 AA 的核心交互要求。
- Seed、模板、导航、奇迹和筛选控件必须具有可访问名称。
- 核心操作必须支持键盘聚焦和执行。
- 自动化组件测试不得出现 axe 可检测的严重无障碍违规。

## 4. 可靠性与错误处理

- 生成入口必须对 Seed、规则版本、模板、干预数组、干预 ID、奇迹类型和目标 ID 做运行时校验。
- 无效领域目标不得静默回退，必须抛出带错误码和字段路径的 `UniverseInputError`。
- 损坏或不支持的分享载荷必须给出警告并安全回退到基础宇宙。
- 同一显式输入必须产生完全一致的结果，任何生成规则变化都必须升级规则版本并更新内容哈希。

## 5. 兼容与发布策略

- 当前规则运行时采用 current-only 策略，不承诺执行旧规则版本。
- 基础分享格式由规则短码标识，干预载荷使用独立版本 `I1`。
- 新分享格式必须支持损坏检测和明确回退，不得把未知格式静默解释为有效输入。
- 发布前必须执行 `npm run check:release`，其中 `npm run check` 验证静态与组件门禁，`npm run test:e2e` 验证 Chromium、Firefox、WebKit 和移动视口。
