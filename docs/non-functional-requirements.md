# 非功能需求

本文件是性能、兼容性、无障碍、可靠性和错误处理的唯一事实来源，阶段规格不再重复定义这些长期要求。

当前涉及一次性生成、固定模板、奇迹和 I1 分享的条目只适用于旧基线。对应模块完成 [产品重构实施路线](milestones.md) 与 [旧产品模块迁移矩阵](legacy-module-migration.md) 中的替换后，必须在本文件中改为新运行时预算和兼容策略，不能继续约束新产品形状。

## 1. 性能预算

- 旧基线普通测试环境中连续生成 50 个静态宇宙应在 2 秒内完成，覆盖率插桩环境允许放宽到 5 秒；步骤 2 后由状态转换吞吐、暂停响应、存档恢复和长时间跨度预算替换该指标。
- 旧 Seed 法则对比只在用户进入法则页面时计算；步骤 4 后由分支比较和明确的跨宇宙比较预算替换该要求。
- 生产构建后的 JavaScript 总压缩体积不得超过 110 KiB，CSS 总压缩体积不得超过 10 KiB。
- 日常构建门禁使用更严格的扩展保护线：JavaScript 不超过 105 KiB，CSS 不超过 8 KiB，为后续版本保留发布余量。
- 任意观察与可视化能力都必须保留可操作的文字路径，不得要求先加载重型可视化运行时。
- 步骤 2 必须为单步转换、连续运行、暂停响应、状态快照和恢复定义可自动验证的性能预算。
- 步骤 7 必须为长期分支数量、因果历史规模、IndexedDB 存储、导入导出和迁移定义容量与响应预算。

## 2. 兼容性

- 支持具备 ES2022、Web Crypto、URLSearchParams 和 Clipboard API 的现代桌面及移动浏览器。
- 页面最低适配宽度为 360 像素，现有响应式断点为 1180 像素与 720 像素。
- 剪贴板 API 不可用时必须提供手动复制回退，不得导致页面崩溃。
- 非法表单输入必须在页面内显示可访问错误，不得让同步生成异常逃逸到渲染层。
- 应用顶层必须提供未预期异常恢复界面。
- 模拟核心不得读取浏览器、网络、本地存储或系统时间状态。

## 3. 无障碍

- 目标为 WCAG 2.1 AA 的核心交互要求。
- 创世条件、运行控制、观察、实验、干预、分支、导航和筛选控件必须具有可访问名称；旧 Seed、模板和奇迹控件在替换前同样适用。
- 核心操作必须支持键盘聚焦和执行。
- 自动化组件测试不得出现 axe 可检测的严重无障碍违规。

## 4. 可靠性与错误处理

- 当前旧生成入口必须继续校验 Seed、规则版本、模板和干预；新运行时必须校验宇宙宪法、状态转换、随机流状态、实验、宇宙内干预、分支、存档和分享输入。
- 无效领域目标不得静默回退，必须抛出带错误码和字段路径的 `UniverseInputError`。
- 损坏或不支持的分享载荷必须明确拒绝、受控迁移或以静态只读方式打开，不得用其他规则生成并冒充原宇宙。
- 完备原因必须产生完全一致的结果；状态相同但历史不同的分支必须保留独立身份与因果记录。

## 5. 兼容与发布策略

- 旧基线规则运行时暂时采用 current-only 策略，基础分享格式由规则短码标识，干预载荷使用 `I1`；该策略必须在步骤 4 和步骤 7 被创世条件包、历史分支包、静态创作档案以及明确的版本迁移策略替换。
- 新分享格式必须支持损坏检测和明确回退，不得把未知格式静默解释为有效输入。
- 新产品中的未知版本只能受控迁移、明确拒绝或作为不可运行静态快照打开，不允许按当前规则重建并声称等价。
- 发布前必须执行 `npm run check:release`，其中 `npm run check` 验证静态与组件门禁，`npm run test:e2e` 验证 Chromium、Firefox、WebKit 和移动视口。
