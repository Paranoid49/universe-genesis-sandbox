## 1. 可运行产品

默认启动进入运行中宇宙。玩家可以单步推进、连续运行、暂停、调整速度、观察持久对象变化、浏览已经发生的历史、查询运行因果、保存检查点、刷新页面、恢复检查点并继续确定性演化。

旧步骤 1 静态宇宙、旧分享和 A1 图书馆只在明确标识的“旧版隔离兼容视图”中按需加载，不进入默认运行事实来源。打开旧分享查询会进入兼容视图，不会自动升级成等价运行历史。

## 2. 实现边界

- `contracts/runtime.ts`：定义独立版本的宇宙定义、状态、时钟、随机流、输入、转换、差异、事件、运行因果、存档和存储适配器契约。
- `runtime-clock.ts`：提供不读取墙上时间的不可变逻辑时钟。
- `runtime-random.ts`：提供可序列化、可恢复且保留精确决定证据的运行随机流。
- `runtime-state.ts`：提供唯一运行事实、确定性转换、只由逻辑世界事实构成的状态身份、完整对象差异、持久对象、输入顺序和转换语义重放校验。
- `runtime-events.ts`：只从已提交状态差异生成事件。
- `runtime-history.ts`：反向投影指定历史时刻对象，不修改当前状态。
- `runtime-causality.ts`：连接宇宙定义、初态、规则、输入、随机决定、差异、转换、后态和事件，支持原因与后果双向查询。
- `runtime-archive.ts`：提供版本化存档、稳定校验和、稳定错误码、转换历史语义重放、完整性验证和恢复后的下一步一致性。
- `runtimeStorage.ts`：完整运行数据使用带迁移入口的版本化 IndexedDB 适配器，测试使用同契约内存适配器；`localStorage` 不保存状态、历史、因果或随机流。
- `useRuntimeUniverseModel.ts`：串行化推进与存档任务，保存或恢复开始时立即暂停，失败后保持可恢复暂停状态，页面切换停止调度，连续运行单次最多自动提交 100 步。
- `runtimeScheduler.ts`：提供运行界面实际使用的可停止调度器，并由非覆盖率性能测试验证暂停后不再安排下一步。
- `RuntimeApplication.tsx`：默认运行产品壳，不依赖旧完整生成器或旧应用模型。
- `LegacyApplication.tsx`：按需装配的步骤 1 隔离兼容视图。

## 4. 质量与威胁模型证据

步骤 2 当前发布快照：模拟测试 126 项；模拟覆盖率 92.53%／83.8%／94.93%／96.19%；UI 测试 66 项；UI 覆盖率 89.72%／82.08%／91.37%／93.02%；独立性能测试 3 项；E2E 62 项通过、2 项跳过；JavaScript gzip 96633 字节；CSS gzip 6080 字节。
步骤 2 当前定向测试组成：步骤 2 核心契约 19 项；步骤 2 架构 3 项；步骤 2 UI 与存储 22 项；步骤 2 性能 2 项；步骤 2 四浏览器纵向闭环 8 项；合计 54 项。

- 模拟核心和 UI 全局覆盖率已高于冻结阈值，状态转换、随机恢复、运行存档和存储适配器设置独立文件阈值。
- 独立非覆盖率性能测试执行两次预热和十二个多模板样本，覆盖单步、一百步、快照与恢复预算，并单独覆盖 1、2、4、8 倍速度的暂停调度预算。
- 存档任务锁阻止保存、恢复和推进重入；连续调度批次达到预算后自动暂停，不跳步。
- JSON 规范化修复覆盖可选字段与负零值，状态发生阶段变化后的存档仍能稳定往返。
- 运行模块不读取 `Date.now`、`new Date`、`Math.random`、浏览器存储或后台计时器。
- 页面离开、暂停、错误与测试结束均释放连续运行调度；发布检查结束还必须核对端口和项目 Node 进程。
