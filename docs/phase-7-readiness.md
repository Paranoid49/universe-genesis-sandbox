# 阶段 7 准入检查

本文件只判断阶段 7 是否具备开始编写和实现规格的条件，不定义阶段 7 的产品功能。

## 1. 准入条件

| 条件 | 当前证据 |
| --- | --- |
| 奇迹具有真实领域效果 | 行星、恒星系、文明与神话对象会产生结构化字段变化，并记录 `TargetMutation` |
| 干预可完整复现 | 分享码和链接会携带版本化 `InterventionInput[]`，重新生成结果完全一致 |
| 输入边界安全 | 非法 Seed、模板、奇迹、重复 ID 和错误目标均由运行时校验拒绝 |
| 模块职责可继续扩展 | 时间线影响、文明 recipe、干预领域修改、分享控制器与目标选择已拆分 |
| 性能风险受控 | 法则对比按页面延迟计算，生成性能与构建体积均有预算门禁 |
| 质量门禁完整 | 已有 Lint、架构、覆盖率、交互、键盘、无障碍、构建和体积检查 |
| 非功能要求明确 | 兼容性、性能、无障碍、可靠性和发布要求由独立文档维护 |
| 阶段边界保持 | 当前源码仍未实现 Canvas、WebGL、Three.js、时间播放或可视化叠层 |

## 2. 阶段 7 实施约束

- 必须先形成独立阶段 7 规格，再新增实现代码。
- 可视化层只能消费结构化投影数据，不得直接调用或改写生成器内部规则。
- 时间播放只能解释已有时间线，不得把播放状态写回确定性宇宙结果。
- 重型可视化必须按页面延迟加载，不能增加基础文字页面的初始执行成本。
- 任何新增投影字段都必须说明来源、稳定 ID、版本影响和降级行为。
- 阶段 0 至阶段 6 的全部门禁必须继续通过。

## 3. 准入结论

准入状态必须以当前提交的 CI `check` 与 `e2e` 作业为准，文档不再保存可能过期的静态通过声明。

两个作业同时通过后，项目才满足进入阶段 7 独立规格设计的条件，且仍不得跳过规格直接实现阶段 7 功能。
