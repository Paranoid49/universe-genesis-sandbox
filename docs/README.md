# 文档导航

本目录只在根层保留当前仍然指导产品与开发的文档。已经完成、被替代或只反映某个历史时点的文档统一放入 [archive](archive/README.md)，不得继续作为新需求和实现决策的依据。

## 当前事实来源

阅读顺序如下：

1. [产品定义](product-definition.md)：定义产品是什么、玩家与宇宙的关系、核心体验和不可违背的产品原则。
2. [产品重构实施路线](milestones.md)：定义从当前工程基线迁移到目标产品的实施顺序与阶段验收条件。
3. [旧产品模块迁移矩阵](legacy-module-migration.md)：逐项规定旧模块保留、增强、降级、替换或删除的步骤与完成证据。
4. [步骤 1：可追因的宇宙生成器](step-1.md)：记录已经完成并通过完整门禁和独立评审闭环的因果契约、兼容边界和验收证据。
5. [步骤 2 冻结验收口径](step-2-freeze.md)：冻结持续演化运行时的需求、非目标、完成条件、兼容边界、质量要求和必要威胁模型。
6. [步骤 2：持续演化的宇宙](step-2.md)：记录当前实现、完成条件证据、非目标保持情况和待关闭事项。
7. [步骤 3 冻结验收口径](step-3-freeze.md)：冻结自由探索观察台的认知边界、研究记录、主流程迁移、完成条件和质量要求。
8. [架构说明](architecture.md)：定义现有代码分层、依赖方向、确定性约束和扩展边界。
9. [质量门禁](quality-gates.md)：定义自动化检查、测试覆盖率和发布门禁。
10. [非功能需求](non-functional-requirements.md)：定义性能、兼容性、可靠性、无障碍和发布要求。
11. [贡献指南](../CONTRIBUTING.md)：定义长期工程原则，并登记仍处于兼容期的旧基线开发约束及其分步骤退出责任。

发生冲突时，产品范围以产品定义为准，实施顺序以产品重构实施路线为准，旧模块去留以迁移矩阵为准，当前代码边界以架构说明为准。
