# 质量门禁

本文件同时服务当前旧基线与新产品迁移。旧产品专属门禁只在 [旧产品模块迁移矩阵](legacy-module-migration.md) 标记的替换步骤完成前有效，完成后必须由对应新契约测试接管，不能继续锁定旧产品形状。

## 1. 唯一完整检查

```text
npm run check
```

完整检查依次执行代码规范、应用与测试类型检查、AST 架构与单元测试、分层覆盖率、浏览器组件交互、生产构建和构建体积预算。

发布级检查还必须执行真实浏览器端到端测试：

```text
npm run check:release
```

## 2. 自动化门禁

| 门禁 | 要求 |
| --- | --- |
| ESLint | TypeScript 推荐规则、React Hooks 规则、禁止显式 `any` 和未使用变量 |
| 架构边界 | 使用 TypeScript AST 解析真实导入、导出、别名调用和浏览器状态访问，不使用文本正则作为主要依据 |
| 文件规模 | 核心编排文件设置行数上限，超过上限必须先拆分职责 |
| 确定性 | 完备原因必须完全复现同一结果；当前基线使用 Seed、模板、规则版本和干预输入，后续步骤使用宇宙宪法、完整状态、随机流状态和有序显式输入 |
| 规则哈希 | 生成规则内容变化必须同步规则版本和测试基线 |
| 覆盖率 | 模拟核心和 UI 分别生成报告，模拟核心语句与行不低于 90%；空间浏览、页面组件、观察投影、存档契约和主 view-model 设置独立阈值 |
| 组件交互 | 当前主流程、观察、实验、干预、分支和存档操作必须通过真实 DOM 交互测试；旧奇迹与九页导航只属于替换前基线 |
| 无障碍 | 核心页面通过键盘聚焦测试和 axe 自动化扫描 |
| 真实浏览器 | Playwright 覆盖 Chromium、Firefox、WebKit、移动视口、当前分享类型恢复、错误恢复、文件或剪贴板回退和真实 axe 扫描 |
| 构建 | TypeScript 与 Vite 生产构建必须成功 |
| 体积 | 发布硬上限为 JavaScript 110 KiB、CSS 10 KiB；日常扩展保护线为 JavaScript 105 KiB、CSS 8 KiB |

## 3. 测试分层

- `tests/phase-*.test.ts` 只验证旧基线兼容行为，其中永久不变量必须迁入新契约测试，固定内容数量和旧页面形状必须按迁移矩阵在对应步骤退役。
- `tests/architecture.test.ts` 验证依赖方向与文件规模。
- `tests/ui/*.test.tsx` 验证浏览器组件交互、键盘与无障碍行为。
- `tests/e2e/*.spec.ts` 验证真实浏览器、移动视口和跨浏览器行为。
- 负向测试必须覆盖当前有效的宇宙宪法、状态转换、实验、干预、分支、存档和分享输入；旧 Seed、模板和奇迹负向用例只保留在兼容路径中。
- 每个新产品步骤必须增加至少一条浏览器纵向闭环测试，并验证该步骤负责的迁移矩阵条目已经退出旧主流程。

## 4. 变更纪律

- 仅修改 UI 或文档时不得改变规则内容哈希。
- 修改 `src/sim/**` 时必须判断是否改变确定性输出。
- 行为变化必须更新适用的规则、状态、存档或分享版本，并同步当前产品步骤、迁移矩阵和兼容策略；不得继续修改归档阶段文档来证明新行为。
- 删除或替换旧行为时，必须先提供新契约测试、迁移或明确拒绝测试，再移除对应旧形状断言。
- 不允许通过降低覆盖率、性能或文件规模阈值来绕过失败，应先修复实现或补充合理证据。

## 5. 本地真实浏览器验证

根据仓库进程管理要求，开发者应在可见终端中手动启动预览服务：

```text
npm run build
npm run preview -- --port 4173
```

在另一个终端执行：

```text
npm run test:e2e
```

测试结束后在预览服务终端按 `Ctrl + C` 停止服务。
