# 阶段 8：本地存档、分享与宇宙图书馆

阶段 8 的目标是在不引入服务端依赖的前提下，为当前浏览器提供可校验、可搜索、可收藏、可导入导出的本地宇宙图书馆。存档只保存复现宇宙所需的显式输入和用户元数据，不保存或修改模拟核心的派生结果。

## 1. 目标

- 将当前宇宙保存到浏览器本地存储。
- 支持标题、收藏状态、创建时间和更新时间等用户元数据。
- 支持按标题、Seed、模板和分享码搜索。
- 支持从存档恢复同一基础宇宙或干预分支。
- 支持版本化 JSON 导出和导入。
- 对损坏、不支持或超容量数据提供明确错误，不静默丢失现有存档。

## 2. 范围与非目标

阶段 8 只实现本地浏览器图书馆，不实现：

- 用户账户、云同步、服务端数据库或公开社区。
- 多人协作、权限、评论、点赞或远程搜索。
- 旧规则运行时；规则兼容策略仍为 current-only。
- 把完整 `UniverseSummary` 快照作为长期真实来源。
- 自动后台保存、定时器同步或网络请求。

## 3. 分层边界

- `src/sim/**` 不得读取 `localStorage`、系统时间、文件或浏览器 API。
- `src/ui/archive.ts` 定义纯存档契约、解析、校验、搜索和不可变更新函数。
- `src/ui/archiveStorage.ts` 是唯一允许访问浏览器本地存储的模块。
- `src/ui/useUniverseArchive.ts` 管理图书馆视图状态、错误和持久化协调。
- `src/components/UniverseLibrary.tsx` 只消费图书馆状态与回调。
- 存档恢复必须回到现有生成入口，不得直接信任持久化派生对象。

## 4. 存档契约

```ts
type ArchiveFormatVersion = "A1"

type UniverseArchiveEntry = {
  id: string
  title: string
  seed: string
  displaySeed: string
  templateId: UniverseTemplateId
  rulesetVersion: string
  shareCode: string
  favorite: boolean
  createdAt: string
  updatedAt: string
}

type UniverseArchiveEnvelope = {
  format: "ugs-universe-archive"
  version: ArchiveFormatVersion
  exportedAt: string
  entries: UniverseArchiveEntry[]
}
```

- `id` 由当前分享码稳定派生，同一可复现分支重复保存时更新原条目。
- `shareCode` 是基础 Seed、模板和阶段 6 干预载荷的版本化复现入口。
- 时间字段仅用于排序和用户元数据，不得进入宇宙生成。
- 存档格式版本与规则版本分离，当前格式为 `A1`。

## 5. 校验与兼容

- 导入前必须校验信封标识、版本、字段类型、条目数量和字符串长度。
- 每个分享码必须通过现有分享解码边界验证。
- 只允许恢复当前 `RULESET_VERSION` 可执行的条目。
- 未知存档版本必须明确拒绝，不得按 `A1` 猜测解析。
- 单次图书馆最多保存 50 条，超限保存或导入必须返回明确错误。
- 导入采用合并策略：相同 `id` 的导入条目更新标题、收藏和时间元数据，不重复创建。
- 导入失败时保持原图书馆不变。

## 6. 图书馆交互

- 主导航新增“图书馆”页面。
- 页面显示当前宇宙保存标题输入与“保存当前宇宙”操作。
- 列表默认按收藏优先、更新时间倒序排列。
- 支持搜索、只看收藏、恢复、收藏/取消收藏和删除。
- 恢复后切换到宇宙概览，并显示对应 Seed、模板和干预分支。
- 删除必须由用户显式执行，阶段 8 不增加额外确认弹窗。
- 空列表、无搜索结果、存储不可用和损坏数据都有可访问提示。

## 7. 导入导出

- 导出生成格式化 JSON 文本，供用户复制或保存。
- 导入通过多行文本框粘贴 JSON，不自动读取文件系统。
- 导出内容必须是完整 `A1` 信封，并包含当前全部存档。
- 导入成功后显示新增或更新数量。
- JSON 解析、结构校验或分享码校验失败时显示具体错误，不覆盖现有条目。

## 8. 存储与可靠性

- 本地存储键固定为 `ugs.universeArchive.A1`。
- 读写异常必须被捕获并转化为页面错误状态。
- 内存状态先通过纯函数计算，持久化成功后才提交到页面状态。
- 不启动后台进程、定时器或监听器。
- 测试通过注入内存存储和时钟完成，不依赖真实用户数据。

## 9. 无障碍与响应式

- 搜索、标题、导入文本、收藏筛选和所有条目操作具有可访问名称。
- 错误使用 `role="alert"`，成功反馈使用 `role="status"`。
- 键盘可以完成保存、搜索、收藏、恢复、删除、导入和导出。
- 360 像素宽度下列表与操作不得横向溢出。

## 10. 验收标准

- 同一分享码重复保存只更新一个稳定条目。
- 存档恢复生成与原条目分享码一致的宇宙和干预分支。
- 搜索覆盖标题、Seed、模板和分享码，收藏筛选可组合使用。
- 列表按收藏优先、更新时间倒序排列。
- 删除、收藏和标题更新不会修改宇宙生成结果。
- `A1` 导出后可无损导入空图书馆。
- 损坏 JSON、未知版本、非法条目、无效分享码和超容量导入均被拒绝。
- 本地存储失败时页面保留原状态并显示明确错误。
- 阶段 0 至阶段 7 的全部测试继续通过。
- 新增阶段 8 单元测试、UI 交互测试和真实浏览器路径。
- `npm run check` 与手动 `npm run check:release` 继续通过。

## 11. 开发顺序

1. 定义 `A1` 存档契约、纯校验和集合操作。
2. 封装本地存储适配器。
3. 增加图书馆状态 Hook 和恢复回调。
4. 增加图书馆页面、搜索、收藏和导入导出交互。
5. 增加阶段 8 自动化验收。
6. 更新架构、里程碑、README 与仓库就绪文档。
