# 架构问题分级与修复记录

本文件记录阶段 8 完成后的严格架构评审、严重级别和修复证据。严重级别只描述对当前阶段和下一阶段的实际风险，不以代码行数或主观偏好单独定级。

## 1. 分级标准

- P0：导致数据损坏、安全边界失效、确定性复现失效、核心流程不可用或无法发布的问题。
- P1：明显阻塞下一阶段扩展、造成主要功能难以使用、形成错误领域结果或破坏架构边界的问题。
- P2：当前可运行但会造成性能、测试盲区、维护成本或发布余量持续恶化的问题。
- 建议：不影响当前阶段正确性和下一阶段启动，可按产品路线择机处理的改进。

### P0

未发现 P0 问题。

| 异步导入可能覆盖导入期间新增或同 ID 收藏、删除、重存 | 已修复 | 导入保存开始快照并采用本地变更优先的三方合并；纯函数测试覆盖同 ID 收藏、删除和重存，UI 测试覆盖异步并发保存与收藏 |

## 3. 建议

- 阶段 9 开始前为扩展包格式增加独立 ADR，记录版本、信任边界和迁移决策。
- 后续如文明样本继续增长，可在现有分页之上评估服务端无关的窗口化渲染，但当前每页 30 项不构成阻塞。
- 公开发布前确认 current-only 规则运行时策略是否继续适用。

## 4. 关闭条件

本轮问题只有在完整 `npm run check:release`、真实桌面与移动界面复核、独立严格架构评审均未报告 P0、P1 或 P2 时才可关闭。

## 5. 最终关闭结论

- 完整 `npm run check` 已通过，领域测试 78 项、UI 测试 26 项全部通过。
- Playwright 四项目共 30 项通过，2 项按非 Chromium 浏览器条件跳过。
- 最终独立严格架构复审综合评分为 8.9/10，结论为 P0、P1、P2 均为零。
- 当前只保留阶段 9 ADR、公开发布兼容策略和未来多标签页并发等建议项，本轮修复循环关闭。
