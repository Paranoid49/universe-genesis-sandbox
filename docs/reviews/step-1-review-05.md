## 4. P1 问题

### P1-01：公共投影 API 是间接认证写入通道

- 公共入口导出 `appendCausalProjections`，调用方可以自行提供主题、节点类型、标签、描述、原因节点和规则节点。
- 投影构建只检查节点存在和结构闭包，不验证描述是否由原因值计算，也不禁止与已有事实重复主题。
- 任意结构合法的调用方投影会由私有闭包认证，随后可以通过公共校验和序列化。
- 架构门禁只按认证函数名称检查导出，不能发现不含认证字样的间接写入入口。

可验证整改条件：

1. 公共层不得接收任意投影规格后返回生产认证图。
2. 受控领域投影器只能接受类型化查询参数，或通用投影返回独立的非生产认证视图契约。
3. 重复事实主题、无关原因、任意描述和错误规则组合不能获得认证或序列化权限。
4. 架构门禁检查所有可到达认证写入闭包的函数，而不是只搜索名称。

### P1-02：随机结果绑定不是独立转录

- `randomResultBindings` 直接从最终节点的 `randomSampleRefs` 生成。
- `expectedRandomBindings` 只是同一数组的 `structuredClone`，正常控制流中必然相等。
- 结构校验再次从待校验图自己的节点引用反推期望绑定，因此同步修改引用和绑定表仍会一致。
- 输出路径统一机械拼为 `universe.${subjectId}`，不是实际领域输出位置。
- 当前同步搬运测试最终被拒绝只是因为克隆没有 WeakSet 身份，不是绑定协议识别了搬运。

可验证整改条件：

1. 期望绑定在节点规格或领域结果产生时独立记录，不能从最终因果节点反向生成。
2. 绑定包含真实领域输出路径或稳定输出身份、决定 ID、作用域、节点语义和绑定类型。
3. 因果装配完成后对独立期望转录、最终节点引用和公开绑定表进行三方比较。
4. 同步修改节点引用、主题、类型、作用域、绑定类型和公开绑定表后，独立期望转录校验必须失败。

## 5. P2 问题

### P2-01：随机参数变异测试矩阵不完整

当前测试覆盖六类操作整体互换，但没有分别覆盖同操作下的范围、概率、候选顺序、候选内容和权重同步修改。

必须为 `range`、`int`、`bool`、`pick` 和 `weighted` 增加同步参数变异，并分别验证结构自洽与可信完整重放拒绝。

### P2-02：非功能文档保留过期体积

`docs/non-functional-requirements.md` 仍登记 JavaScript gzip 109556 字节，而当前完整门禁和其他当前文档均为 110551 字节。

当前事实来源必须统一最新体积，历史数字只能保留在带轮次标识的历史评审报告中。

## 6. 已确认关闭事项

- 受控工厂会重新执行完整领域生成并核对首次领域结果。
- 完整随机调用转录在构图前独立克隆，并与最终图逐字段比较。
- 六种随机操作均具有类型化参数和内部语义校验。
- 文明候选负事实只引用生物圈形成、文明概率和魔法适应三个前置作用域。
- `ugs-causality@5`、`ugs-random-bindings@1`、模拟核心哈希和规则版本在代码与主要当前文档中一致。
