## 5. P1 问题

### P1-01：影响可见结果的随机决定没有进入因果图

证据如下：

- `src/sim/timeline.ts` 使用 `timeline:count` 决定时间线事件总数，而 `src/sim/causality-history.ts` 只引用逐事件作用域。
- `src/sim/galaxies.ts` 使用 `root.galaxies` 的抽样决定星系总数，而 `src/sim/causality-world.ts` 只引用逐星系子流。
- `src/sim/civilizations.ts` 为每个文明决定历史事件数量，而 `src/sim/causality-world.ts` 只引用逐事件作用域。
- 现有测试只证明已经存在的随机引用有效，没有反向证明每个影响可见结果的决定至少被一个因果节点引用。

独立动态审计以 Seed `AUDIT-TRACE-001`、模板 `high_magic` 生成宇宙，得到 40769 次随机决定、40190 个随机引用和 579 次未引用决定，其中 576 次为文明历史数量决定，另外包括时间线数量、星系数量和一次未产生生物圈的抽样决定。

影响如下：

时间线事件数、星系数和文明历史事件数是玩家可见结果的一部分，但当前原因链无法说明集合边界为什么是这个数量，违反“所有果都能找到对应的因”。

可验证整改条件如下：

1. 为集合或聚合结果建立明确因果节点，或者把数量决定挂接到所有受其存在性影响的成员节点。
2. 至少覆盖时间线数量、星系数量和文明历史数量。
3. 明确未产生对象的负事实是否属于步骤 1 可见结果，若属于则建立可查询结果节点，若不属于则在产品和步骤文档中明确边界。
4. 新增多 Seed、多模板和有无干预场景的反向随机决定审计。

### P1-02：输入清单可以在重算身份后与实际干预事实脱节

证据如下：

- 生成身份是输入清单内容的可重算散列。
- 校验器核对清单 ID、随机追踪 ID、顺序与根集合，但没有核对干预根中的实际类型化载荷。
- 干预根 `subjectId` 只有干预 ID，实际 `miracleType -> targetId` 只存在于描述文本中。
- 当前值篡改测试没有同步重算生成身份，因此没有覆盖整体一致伪造。

独立篡改把真实干预清单从 `bless_planet/真实目标` 改为 `erase_memory/forged-target`，同步重算生成身份与随机追踪生成 ID 后，原图仍被校验为有效。

影响如下：

序列化图和未来存档不能证明自己对应哪组显式输入，首轮输入根真实性问题没有完全关闭。

可验证整改条件如下：

1. 输入根保存不可歧义的类型化规范载荷，或校验入口携带可信生成上下文。
2. Seed、规则版本、模板以及每项干预的 `kind/order/subjectId/value` 与图中对应事实一一核对。
3. 新增篡改输入值、同步重算清单 ID 与随机追踪生成 ID 后仍被拒绝的负向测试。
4. 覆盖同 ID 不同奇迹类型、同 ID 不同目标以及同时伪造根说明与清单值的受信边界。

## 6. P2 问题

### P2-01：随机追踪没有验证操作、候选集和选中值语义

证据如下：

- 随机运行时为 `next`、`range`、`int`、`bool`、`pick`、`weighted` 定义了不同候选语义。
- 校验器只检查决定结构、候选集标识非空以及引用字段相等，没有重新验证操作语义。
- 同时篡改决定与全部引用后，伪造候选集和不属于候选集的选中值仍能通过校验。

影响如下：

损坏或伪造的随机证据可以被报告为有效，玩家看到的候选集合与最终选择不一定来自一次合法随机操作。

可验证整改条件如下：

1. `operation` 必须属于受支持枚举，并按操作类型验证候选与结果语义。
2. `pick` 和 `weighted` 验证选中项属于候选集合，`bool`、`int`、`range`、`next` 验证类型与范围。
3. 候选集标识必须由规范候选载荷重新计算。
4. 新增同时篡改决定和全部引用的未知操作、伪造候选标识、非法选中值与权重不一致负向测试。
