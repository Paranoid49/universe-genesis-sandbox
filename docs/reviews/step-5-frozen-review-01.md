评审对象：步骤 5“多本体宇宙沙盒”及其对步骤 1 至步骤 4 的直接回归喵。

## 2. 阻塞问题

### P1-01｜拓扑模块没有形成运行时关系或图状态

- 严重等级：P1 喵。
- 涉及位置：`src/sim/contracts/constitution.ts:96`、`src/sim/contracts/runtime.ts:44`、`src/sim/runtime-state.ts:34`、`src/components/ConstitutionCreator.tsx:67` 喵。
- 触发或复现方式：分别选择物质、奥术和梦流预设创建宇宙并检查运行状态，三个预设虽然显示 `hierarchical`、`relational`、`semantic` 及关系名称，但 `RuntimeWorldObject` 只有属性，没有关系、边或等价拓扑结构，初始状态创建也只按对象类型各创建一个孤立对象喵。
- 实际影响：三个预设的拓扑差异只存在于创世摘要，运行、演化、观察、分支和存档中的世界结构本质上都是同一种扁平对象表，无法证明统一运行时支持图关系或非传统拓扑喵。
- 判断依据：冻结基线 2.3 明确要求“运行对象的关系和拓扑必须由宇宙宪法决定”以及“通用状态必须支持图关系或等价的非传统拓扑”，2.6 还要求三种拓扑或对象模式能够在产品界面中被实际观察喵。
- 整改建议：为通用状态增加版本化关系或等价拓扑契约，由拓扑模块定义允许的关系类型和初始结构，由统一转换、差异、事件、观察、因果、分支和存档路径共同处理，并增加三个参考预设及组合夹具的真实结构断言喵。

### P1-02｜认知模块声明没有约束公开公理、隐藏字段和观察结果

- 严重等级：P1 喵。
- 涉及位置：`src/sim/contracts/constitution.ts:101`、`src/sim/constitution-validation.ts:115`、`src/sim/observation-access.ts:25`、`src/sim/observation.ts:20` 喵。
- 触发或复现方式：创建一个合法认知模块，将 `publicAxiomIds` 设为空并把某个可观察字段加入 `hiddenAttributeIds`，替换物质预设的认知模块后仍可通过宪法校验；`createObservationAccess` 仍会把全部运行规则列为公开公理，观察方法仍会读取被声明隐藏的字段喵。
- 当前参考预设中的直接证据：共享认知模块声明 `publicAxiomIds: []`，但 `createObservationAccess` 无条件执行 `state.rules.map(...)`，因此页面在玩家取得证据前仍显示全部规则为“公开公理”喵。
- 实际影响：宪法认知边界只是未生效的元数据，合规规则包无法隐藏规则或字段，动态观察可以越过宪法声明读取本应不可见的信息，直接回归步骤 3 的有限认知边界喵。
- 判断依据：冻结基线 2.3 要求认知模块定义可见性与知识边界，2.4 要求观察不得绕过步骤 3 认知边界，威胁模型明确列出“认知边界泄漏”喵。
- 整改建议：校验公开公理和隐藏字段引用，按认知模块过滤公开公理、可见对象、可用观察方法和可读字段，禁止观察方法引用隐藏字段或要求显式授权，并补充可信重算与研究恢复的负向测试喵。

### P1-03｜边界预算既未完整校验，也未完整执行

- 严重等级：P1 喵。
- 涉及位置：`src/sim/contracts/constitution.ts:157`、`src/sim/constitution-validation.ts:115`、`src/sim/constitution-executor.ts:32`、`src/sim/constitution-executor.ts:101`、`src/sim/runtime-state.ts:34` 喵。
- 触发或复现方式：构造 `maximumRuleEvaluations` 或 `maximumEffectsPerStep` 为 `Infinity`、非正数或非安全整数的边界模块，或把 `maximumObjects` 设为零，当前宪法校验不会返回问题；初始状态创建也不会检查 `maximumObjects`，执行器只在两个乘积或差异数量比较点使用其中两个字段喵。
- 实际影响：通过校验的宪法可以声明无效或无界预算，`maximumObjects` 完全不生效，无法阻止对象创建或未来关系扩展造成的单步资源耗尽；非有限数值还会被当前稳定序列化按 JSON `null` 语义处理，削弱内容身份的唯一性喵。
- 判断依据：冻结基线 2.2 明确要求规则执行设置预算以阻止无限规则链、效果展开、递归和资源耗尽，威胁模型明确列出对象创建、关系扩展和因果路径无界增长喵。
- 整改建议：对所有边界字段执行有限、安全整数和正范围校验，在初始创建与每次转换提交前统一执行规则、效果、对象、关系和因果扩展预算，并保证预算失败不消费随机流或提交任何记录喵。

### P1-04｜动态实验与干预仍受固定正负二十和首对象绑定

- 严重等级：P1 喵。
- 涉及位置：`src/sim/branch-inputs.ts:26`、`src/sim/branch-inputs.ts:36`、`src/sim/branch-inputs.ts:58`、`src/components/pages/ExperimentPage.tsx:9`、`src/components/pages/ExperimentPage.tsx:31` 喵。
- 触发或复现方式一：定义允许 `-30` 至 `30` 的合法干预能力，页面会按能力显示该范围，但提交 `25` 时 `inputAdjustments` 仍因固定 `Math.abs(delta) > 20` 拒绝喵。
- 触发或复现方式二：定义多个对象类型，并让干预能力只适用于非首对象类型；`createInterventionInput` 和实验页面都固定选择 `Object.values(state.objects)[0]`，玩家无法选择宪法允许的实际目标喵。
- 实际影响：新增合规规则包不能仅通过声明获得新的参数范围和目标能力，必须修改公共输入实现或调整对象顺序，违反动态能力和扩展边界喵。
- 判断依据：冻结基线 2.4 明确要求实验字段、干预能力、目标类型、参数、约束和代价由宪法声明，2.5 要求页面消费动态契约，C8 和 C9 要求能力随宪法变化且能够正确提交接受或拒绝结果喵。
- 整改建议：从宪法的实验与干预声明生成目标列表、字段和参数约束，公共创建函数显式接收 `objectId` 与 `capabilityId`，移除固定正负二十限制，并让实验与干预分别依据各自声明校验喵。

### P1-05｜多规则随机决定的因果网络会伪造额外规则原因

- 严重等级：P1 喵。
- 涉及位置：`src/sim/runtime-causality.ts:164` 至 `src/sim/runtime-causality.ts:173`，以及 `src/sim/runtime-causality.ts:186` 喵。
- 触发或复现方式：创建一个合法作用模块，包含两个写入不同字段且都使用随机效果的规则，使二者在同一转换中同时应用；`addTransitionNodes` 对每一个随机决定都使用整条转换的 `transition.ruleIds` 作为直接原因，并连接全部已应用规则，而不是该决定所属执行记录的唯一规则喵。
- 实际影响：从任一随机决定反向查询会看到没有生成该抽样的其他规则，因果图虽然通过结构闭包校验，却包含语义错误的因果边；多规则包越丰富，伪造关系越多喵。
- 判断依据：冻结基线 2.2 要求确定性选择形成稳定可重放执行记录且每个差异引用适用随机决定，2.7 要求规则、随机决定、差异和转换能够准确双向追溯，步骤 1 至步骤 4 的因果闭包核心不变量也要求来源真实而非仅结构可达喵。
- 整改建议：由每条 `RuleExecutionRecord.randomDecisionIds` 建立随机决定到具体规则和对象执行的唯一归属，随机节点不得连接整条转换的全部规则，并增加多规则、多对象和同规则多执行实例的语义因果测试喵。

## 3. 非阻塞问题

### P2-01｜动态对象标签泄露内部类型 ID

- 严重等级：P2 喵。
- 涉及位置：`src/sim/observation-access.ts:21` 喵。
- 触发或复现方式：进入观察页查看对象选择器，标签由 ```${object.kind}｜${object.id}``` 生成，会显示 `matter-region`、`arcane-knot` 或 `dream-motif`，而不是本体模块声明的“物质区域”“奥术结点”“梦境意象”喵。
- 实际影响：用户界面泄露内部稳定 ID，新增未知类型没有中文显示名解析路径，降低易用性并违反标签层边界喵。
- 判断依据：冻结基线 2.5 要求动态对象使用规则包声明的中文显示名称，未知类型不能泄露内部枚举喵。
- 整改建议：从本体模块按 `kind` 解析声明名称，找不到时显示明确的“未知对象类型”错误或受控占位，不直接回退内部 ID 喵。

### P2-02｜评审前体积事实快照与同工作树实测不一致

- 严重等级：P2 喵。
- 涉及位置：`docs/step-5.md`、`docs/quality-gates.md`、`docs/non-functional-requirements.md` 和 `docs/legacy-module-migration.md` 的步骤 5 快照喵。
- 触发或复现方式：本次在当前工作树运行完整 `npm run check:release`，构建和体积门禁实测 JavaScript gzip 为 121362 字节，文档记录为 121348 字节；CSS 均为 6492 字节喵。
- 实际影响：不影响 125 KiB 保护线和 130 KiB 硬上限，但“同一份代码上的最终事实快照”不精确，降低发布证据的可审计性喵。
- 判断依据：冻结基线 C16 和质量文档要求完整门禁证据，步骤文档自称记录当前实现与验证事实喵。
- 整改建议：确认体积工具是否存在环境波动；若结果应确定，则以最终关闭评审所用工作树重新生成并同步四份当前事实文档喵。

## 6. 最终关闭判断

当前不允许关闭步骤 5 喵。

阻塞原因是 P1-01 至 P1-05 均具有现实触发路径并直接违反冻结基线，现有完整发布门禁没有覆盖这些组合场景喵。

整改后应由未参与实现和本次评审的全新独立 Agent，只复查尚未关闭问题、本轮修改及其直接回归，不得重新开放式提高标准喵。
