## 1. 整改结论

第十二轮五项问题均已完成根因整改并通过完整 `npm run check:release`。本报告只证明冻结前历史问题已关闭，不替代冻结后的首次独立完整评审。

## 2. P1-01 加权选择证据不完整

- 违反条件：R3、R6、C4。
- 根因：对象候选标签只保留少量身份字段，文明路径还会预先删除不合格候选，导致完整候选顺序、权重和资格无法复核。
- 整改：对象候选标签记录全部字符串、数字与布尔原始字段；文明路径保留全部候选，不合格候选权重为零，并记录 `eligible` 与稳定资格表达式；状态值投影展示类型化参数、原始样本、候选快照、候选顺序、权重和选中值。
- 验证：五类领域加权选择正向测试覆盖星系、恒星系、行星、文明路径和神话类型；资格改写、候选顺序、内容、权重、操作参数和随机流错配均有负向验证。
- 契约：因果契约升级为 `ugs-causality@12`，随机绑定契约升级为 `ugs-random-bindings@5`。

## 3. P1-02 可见结果可能漏登或绑定错误主题

- 违反条件：R3、C1。
- 根因：旧结果契约只登记主题和追因策略，没有保存实际原始值，架构门禁也没有保护宇宙身份、对象名称、Seed 和分享结果。
- 整改：`ResultValue`、`StatTile`、`AttributeBar` 和 `RegisteredResult` 统一输出稳定原始值指纹；受保护页面禁止裸渲染关键派生值；筛选后的时间线数量明确保留为纯界面状态，不再冒充 `timeline.count` 宇宙事实。
- 验证：UI 测试逐项核对状态值指纹与 `stateValueResult` 生产事实一致，并核对宇宙名称、描述、Seed、分享码、分享链接和对象名称的主题与真实值；架构测试保留裸值负向夹具。

## 4. P1-03 统计深层错组和错判不可检测

- 违反条件：R3、C1、C2。
- 根因：既有校验只比较顶层统计描述与直接原因集合，没有复核字段值、成员分组、筛选谓词和纳入结果。
- 整改：新增独立高风险谓词节点，记录版本、字段、运算符和阈值；深层校验按主题索引验证字段值、成员、分组、谓词、纳入结果与精确原因集合。
- 验证：负向矩阵覆盖错误字段值、错误分组键、重复成员、阈值改写、纳入／排除反转和字段来源断开。
- 性能：深层校验使用一次性主题索引与线性集合比较，避免逐成员全图扫描。

## 5. P2-01 当前文档保留冲突发布数字

- 违反条件：R11、C10。
- 根因：当前质量文档同时保留第十轮、第十一轮和当前快照数字，无法确定唯一发布事实。
- 整改：历史数字只保留在对应评审整改报告；当前文档只保留一套完全一致的当前发布快照；架构测试不再把历史整改报告当作当前事实来源。

当前发布快照：模拟测试 104 项；模拟覆盖率 92.58%／83.86%／94.62%／95.88%；UI 测试 43 项；UI 覆盖率 89.96%／81.83%／92.3%／92.76%；独立性能测试 1 项；E2E 54 项通过、2 项跳过；JavaScript gzip 86408 字节；CSS gzip 5809 字节。

## 6. P2-02 普通环境性能预算边缘失败

- 违反条件：R11、C11。
- 根因：单次冷样本同时受启动、转换和主机调度噪声影响，不能稳定表达普通运行性能分布。
- 整改：新增独立非覆盖率性能门禁，执行两次预热后测量十二个多模板样本，P50 与 P90 均必须小于 1500 毫秒；覆盖率环境保留代表性完整路径 3000 毫秒预算。
- 验证：`npm run test:performance` 已纳入 `npm run check:release` 并通过。
