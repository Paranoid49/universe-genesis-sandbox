## 4. P1 问题

### P1-01：随机引用仍可同步搬运到无关结果

- 当前校验只比较引用中的 `resultSubjectId` 与所在节点主题，两者都来自待校验图。
- 把真实生物圈随机引用移动到 `share.code`，并同步修改结果主题后，结构校验仍然通过。
- 文明候选负事实引用整个生物圈命名空间，把结论形成后的主导形态和复杂度抽样倒挂为原因。

可验证整改条件：

1. 受控生成过程产生独立结果绑定转录，保存决定、结果主题、节点类型、绑定类型、领域输出路径和作用域。
2. 认证时使用受控重放产生的绑定转录核对最终图，不接受待校验图自报的期望值。
3. 集合数量、集合成员、普通字段和负事实使用不同绑定类型。
4. 负事实只引用决定该结论的前置样本。
5. 增加同步修改结果主题、节点类型、作用域和绑定记录后的引用搬运负向测试。

### P1-02：认证状态可由普通代码直接赋予

- `causality-certification.ts` 公开导出无条件写入 `WeakSet` 的 `certifyCausalGraph`。
- 修改后的克隆只要调用该函数，就能重新通过公共校验、序列化、投影和比较。
- 测试夹具也直接调用生产认证写入函数，模糊了结构测试与可信生产认证的边界。

可验证整改条件：

1. 认证写入能力不得从任何普通生产模块、测试夹具或扩展代码可直接导入的位置导出。
2. 受控工厂必须重新执行完整领域生成、完整随机调用转录和结果绑定核对后才能认证。
3. 公共序列化、投影和比较只接受运行时已认证图。
4. 普通克隆、同步修改图和重新冻结副本均无法自行获得认证。
5. 测试夹具只使用结构校验 API，不得冒充可信生产图。

## 5. P2 问题

### P2-01：随机重放只证明裸样本，没有证明实际调用操作

- 当前实现能够根据 Seed、命名空间和样本序号重放 `sampleValue`。
- 操作、范围、概率、候选、权重和选中值仍由待校验记录自身提供。
- 保持原始样本不变，把记录整体转换为另一种合法操作并同步更新参数后，六种操作均可通过结构校验。

可验证整改条件：

1. 受控工厂重新执行完整宇宙生成并取得可信决定转录。
2. 最终图逐项核对流、样本、操作、作用域、候选、权重和选中值。
3. 候选和权重保留无歧义、类型化的规范表示。
4. 增加六类操作互换、同操作参数修改、候选重排、候选替换和权重修改负向测试。
