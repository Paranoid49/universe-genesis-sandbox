## 2. P1 整改

### P1-01：随机决定没有完整进入因果图

- 新增集合边界节点，分别保存时间线事件数量、代表性星系数量和每个文明历史事件数量的决定与实际数量。
- 每个集合成员把对应集合边界作为直接原因，数量决定不再孤立于成员结果之外。
- 对没有形成生物圈的行星建立明确负事实节点，保存阈值判断使用的随机证据。
- `validateCausalGraph` 反向遍历全部随机决定，任何未被有效因果引用覆盖的决定都会产生 `INVALID_RANDOM_REFERENCE`。
- 新增多 Seed、多模板和有无干预场景测试，逐一证明全部决定均至少进入一个因果节点。

关闭状态：已整改。

### P1-02：输入清单可在重算身份后与实际事实脱节

- 因果契约升级为 `ugs-causality@3`，输入根保存类型、顺序、主题和值组成的类型化输入证据。
- 干预结果节点同时保存对应输入证据，独立图校验要求清单、输入根和唯一干预节点三者一致。
- `validateCausalGraph` 与 `assertCausalGraph` 支持传入可信期望生成清单；生产构建使用受控生成信封执行额外核对。
- 新增同步篡改清单并重算生成 ID、修改根说明、修改根输入证据、修改干预节点证据和可信清单不一致的分层负向测试。

关闭状态：已整改。

## 3. P2 整改

### P2-01：随机操作与候选语义没有校验

- 随机追踪不再丢弃较大的候选集合，候选集标识可由完整规范候选载荷重新计算。
- `next`、`range`、`int`、`bool`、`pick` 和 `weighted` 分别验证操作、范围、类型、候选身份、权重与选中值。
- 未知操作、伪造候选集标识、候选外选中值、无有效权重和引用同步篡改均不能通过随机追踪校验。
- 随机语义校验拆分到独立模块并进入架构规模门禁。

关闭状态：已整改。
