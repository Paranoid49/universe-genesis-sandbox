## 2. 逐项复查

### P1-01 自主行动拒绝没有统一满足稳定记录且不消费随机

- 处置：关闭，等级维持原 P1 作为历史记录喵。
- 现实触发复查：随机效果越过约束时，`src/sim/constitution-executor.ts:69-91` 在采样前建立检查点，并在约束拒绝时回滚随机流与决定记录；自主行动预算耗尽时，`src/sim/constitution-executor.ts:49-52` 形成稳定 `arbitration-rejected` 执行记录而不中止转换喵。
- 实际影响复查：原先会改变后续随机序列或丢失预算拒绝事实的两条路径均已消除，拒绝动作不产生状态差异、随机消费或部分提交喵。
- 判断依据：冻结 C6 与第 2.4 节要求拒绝保留稳定原因且不消费随机或伪造成功后果喵。
- 验证证据：`tests/step-6-autonomy.test.ts` 的随机约束拒绝与单行动预算测试通过，步骤 5 执行器定向回归同时通过喵。

### P1-02 多对象执行同一规则时状态差异连接到错误主体

- 处置：关闭，等级维持原 P1 作为历史记录喵。
- 现实触发复查：`src/sim/runtime-causality-build.ts:157-166` 使用 `ruleId + objectId + applied` 选择执行记录，并使用行动自身的差异索引补充主体行动原因喵。
- 实际影响复查：潮生两个主体执行同一规则时，每项差异只连接所属对象的执行记录与行动，不再发生跨主体因果污染喵。
- 判断依据：冻结 C5 和因果伪造威胁模型要求无关主体与执行记录不得进入原因集合喵。
- 验证证据：`tests/step-6-autonomy.test.ts:102-116` 逐项核对差异对象、主体、行动和执行记录，定向测试通过喵。

### P1-03 合法的小记忆容量生成不可继续和不可保存状态

- 处置：关闭，等级维持原 P1 作为历史记录喵。
- 现实触发复查：`src/sim/autonomy.ts:72-76` 先以本步实际产生的感知记忆建立信念历史引用，再独立淘汰当前记忆窗口；`src/sim/autonomy-validation.ts` 从完整转换历史校验信念来源喵。
- 实际影响复查：`memoryCapacity = 1` 时主体当前窗口保持一条记忆，但信念仍引用合法历史来源，连续推进、保存、恢复和恢复后下一步均可完成喵。
- 判断依据：冻结 C7 和第 2.3 节要求有限记忆淘汰稳定、来源合法且可恢复运行喵。
- 验证证据：`tests/step-6-autonomy.test.ts` 的容量为一、多感知、连续恢复测试通过喵。

### P1-04 分支状态哈希与状态比较没有覆盖自主状态

- 处置：未关闭，维持 P1 喵。
- 已完成部分：`src/sim/branch-validation.ts:13-15` 已把拓扑与完整当前自主状态纳入状态哈希，修改主体名称或记忆会改变状态哈希喵。
- 仍可复现路径：克隆一个有效潮生状态，只修改一条主体记忆摘要后调用 `compareBranchObjects`，结果仅得到 `field = autonomy.entity.state`、`before = 左侧自主状态`、`after = 右侧自主状态` 喵。
- 涉及位置：`src/sim/branch-comparison-evidence.ts:68-78` 只按实体、关系、叙述和神话档案集合做整对象 JSON 比较，并以固定占位文字输出；`src/components/pages/BranchesPage.tsx:36-39` 原样显示该占位结果喵。
- 实际影响：玩家只能知道“某个实体集合项不同”，无法知道差异来自生命周期、记忆、信念、意图还是行动，也无法在页面看到被修改的认知内容或对应历史差异；原关闭条件要求的“显示具体自主认知差异”仍未满足喵。
- 判断依据：冻结 C10 明确要求分支比较显示自主行动和认知历史差异，接口边界还要求比较覆盖自主行动与认知历史；首次评审的原关闭条件明确要求仅自主认知不同的共同祖先分支在比较结果和页面显示具体差异喵。
- 验证证据：咪使用一次性临时测试复现，仅修改记忆摘要后比较结果恰好只有上述集合级占位项，且结果 JSON 不包含修改后的记忆；临时测试通过后已删除，没有留在工作区喵。
- 整改建议：对实体身份、生命周期、当前记忆窗口、信念、意图、最后行动、关系、公开叙述和神话档案分别生成稳定、可读且不泄漏私有内容的差异类别，并为转换中的自主行动与认知历史提供明确比较证据；页面至少应能区分发生变化的自主语义类别喵。
- 原问题关闭条件：修改任一自主字段会改变状态哈希，且仅自主认知不同的共同祖先分支能在比较结果和页面显示具体差异，同时当前认知相同但历史不同的分支仍保持不同历史哈希喵。

### P1-05 行动因果页面可递归读取私有认知和底层真相

- 处置：未关闭，维持 P1 喵。
- 已完成部分：`src/components/pages/AutonomousEntitiesPage.tsx:38-50` 已将感知、记忆、信念、意图、规则裁决、输入和随机节点显示为公开稳定标签或“内部依据”，不再直接显示原评审中的规则 ID、置信度、感知值和执行枚举喵。
- 仍可复现路径：进入潮生自主实体页面，打开任一已执行行动的因果，点击“状态变化”，页面会直接显示类似 `2 → 1` 或 `0 → 2` 的底层对象属性精确差异喵。
- 涉及位置：`src/sim/runtime-causality-build.ts:157-166` 把 `difference.before` 与 `difference.after` 写入节点描述；`src/components/pages/AutonomousEntitiesPage.tsx:42-50` 虽把差异标签改为“状态变化”，但 `publicCausalDescription` 没有脱敏 `difference`，因此精确底层值仍传给玩家页面喵。
- 实际影响：玩家无需取得对应观察方法或观察授权，即可沿行动后果读取底层状态精确值，继续绕过步骤 3 的有限观察投影；公开标签整改只隐藏了字段名，没有隐藏底层真相喵。
- 判断依据：冻结 C11 和第 2.7 节要求玩家只能看到观察能力允许的行为证据，不得自动看到隐藏感知、私有记忆、完整信念或宇宙底层真相；界面泄漏威胁模型明确覆盖因果描述喵。
- 验证证据：咪使用一次性 UI 临时测试真实点击“查看行动因果”与“状态变化”，确认自主行动因果区域出现精确数值差异；临时测试通过后已删除，没有留在工作区喵。
- 现有测试缺口：`tests/step-6-autonomy.test.ts:127-148` 只匹配若干内部 ID 与关键词，未检查底层数值；`tests/ui/autonomous-entities-page.test.tsx` 只打开和关闭因果，没有下钻到状态差异喵。
- 整改建议：为自主页面建立受 `ObservationAccess` 约束的独立因果投影，未被授权的差异只显示“已产生可观察后果”或等价稳定说明；只有已经通过观察系统公开的值才能显示精确前后差异喵。
- 原问题关闭条件：递归遍历全部行动因果节点时不出现私有认知、内部标识、执行枚举或底层真相，同时授权公开证据仍可双向查询并返回原实体上下文喵。
