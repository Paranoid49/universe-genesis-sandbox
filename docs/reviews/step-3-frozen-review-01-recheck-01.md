## 4. 尚未关闭的 P1 问题喵

### P1-04：概览缺少“近期可观察变化”投影喵

- 涉及位置：`src/components/RuntimeSession.tsx:17-22`、`src/ui/runtimePageModel.ts:4-9`、`src/components/pages/RuntimePage.tsx:14-22` 喵
- 现实触发方式：执行任意结构或规律观察后返回“当前宇宙”，概览把最后四条任意观察信号全部放入“当前可知状态”，没有单独识别近期变化，也没有在没有变化观察时呈现“尚无近期可观察变化”喵
- 实际影响：静态结构结论、规律证据和变化信号被统一当作 `recentSignals`，玩家无法从概览区分当前已知状态与近期发生或未发生的可观察变化喵
- 判断依据：冻结 R8 明确要求新概览由当前可知状态、近期可观察变化、未知区域和玩家关注内容组成，原 P1-04 也把缺少近期可观察变化列为未关闭原因喵
- 根因：概览模型只有通用 `recentSignals`，没有按变化观察或等价语义形成独立只读投影喵
- 整改建议：在不读取底层差异的前提下，从已获得的变化类信号形成独立近期变化投影，并为没有相关信号的状态提供不泄露事实的空状态喵

### P1-05：恢复研究记录后条件对象视图不可进入喵

- 涉及位置：`src/components/pages/ObservationPage.tsx:22-23`、`src/components/pages/ObservationPage.tsx:66`、`src/ui/useObservationWorkbench.ts:18-37` 喵
- 现实触发方式：对传统空间对象完成一次观察并确认条件视图出现，刷新页面使研究记录从存储恢复，再进入观察页喵
- 实际影响：恢复后的 `notebook.signals` 已包含相关对象信号，但 `workbench.signal` 初始仍为 `undefined`，`observedObject` 因此为空，条件视图不会出现，玩家也没有从历史信号重新选择该对象进入条件视图的入口喵
- 判断依据：冻结 R11 要求尚未获得相关信号时隐藏条件视图，C10 要求传统空间对象可以通过观察上下文进入条件视图，已经持久化并恢复的信号不能被当作“尚未获得”喵
- 根因：条件视图的对象身份依赖本次挂载期间最后一次新观察，而不是研究记录中已经获得的可见信号或显式历史选择喵
- 整改建议：让条件视图从已恢复的信号集合和玩家显式选择得到对象上下文，同时继续确保没有信号时入口、路由和可访问名称均不存在喵

### P1-06：重算内层身份后仍可伪造观察结果与来源喵

- 涉及位置：`src/sim/observation-identity.ts:6-31`、`src/sim/research-archive.ts:56-78`、`tests/step-3-observation.test.ts:216-233` 喵
- 现实触发方式：取得合法信号后修改 `visibleValue`、证据摘要、`sourceSubjectIds` 和 `causalNodeIds`，随后分别调用公开的 `evidenceIdentity` 与 `signalIdentity` 重算证据和信号身份，再调用 `createResearchArchive` 生成新的外层校验和喵
- 独立验证结果：咪用临时定向测试把来源改为 `forged-subject`、把因果节点改为不存在的 `runtime-cause:transition:forged`、把可见结论改为“伪造的观察结论”，同步重算全部内外层身份后，`createResearchArchive` 未抛错并生成合法信封喵
- 实际影响：本地篡改、迁移错误或错误调用方可以把不存在的结论和因果来源重新封装为完整性校验通过的研究存档，随后界面会把它展示为真实观察证据喵
- 判断依据：冻结 R2、C4、5.2 身份边界和“证据来源伪造”威胁模型要求每个结果定位到实际状态或已提交历史，并明确拒绝篡改来源喵
- 根因：`validateEvidence` 只要求来源主题包含对象 ID、因果节点数组非空且内容哈希自洽，`validateObservableSignal` 只要求信号内容哈希自洽，二者都没有与可信状态、已提交转换或受控观察证明进行语义核对喵
- 现有测试缺口：所谓“重新封装篡改”测试只修改字段但保留旧内层 ID，因此证明的是陈旧哈希会被拒绝，没有覆盖攻击者同步重算内层 ID 的原评审触发路径喵
- 整改建议：存档创建与解析必须接收或携带可验证的受控观察证明，至少能核对状态身份、方法、对象、时刻、可见结果和每个来源节点确实属于对应状态或已提交历史，不能把公开可重算内容哈希当作来源真实性证明喵
