## 4. P1 问题

### P1-01：随机决定只要求被某处引用，没有证明被正确结果引用

- 校验器只收集有效引用的决定 ID，并检查是否存在未引用决定，不核对决定作用域与节点主题、节点类型或领域目标。
- 把一个 `negative_fact` 的真实随机引用移动到无关的 `share_result/share.code` 节点后，图仍通过校验。
- 时间线集合边界把干预追加事件计入总数，但原因中只有基础数量决定，没有干预输入、干预节点或追加事件。

可验证整改条件：

1. 随机引用保存并校验真实结果主题、集合身份或输出字段路径。
2. 校验器核对决定作用域和引用节点之间的允许映射。
3. 数量决定绑定集合身份，成员与聚合节点通过明确关系引用集合边界。
4. 时间线区分基础生成数量和干预追加数量，最终边界包含全部追加来源。
5. 增加引用搬运、节点交换和错误集合挂接负向测试。

### P1-02：公共校验与序列化入口接受同步伪造图

- 带可信期望清单的生产校验能够拒绝伪造，但该参数是可选的。
- `serializeCausalGraph`、投影和比较入口使用无可信上下文的默认断言。
- 同步修改清单、输入根、干预节点、生成 ID 与随机生成 ID 后，默认校验和序列化均成功，只有额外传入原始可信清单时才拒绝。

可验证整改条件：

1. 拆分结构一致性校验与可信生成认证 API、结果类型和状态。
2. 序列化、投影、比较与未来存档不得把只通过内部一致性校验的图视为可信图。
3. 可信模式要求来自待校验图之外的生成上下文或受控重放结果。
4. 增加对序列化、投影和比较入口的整体同步伪造负向测试。

### P1-03：可见的“未达到文明候选门槛”没有因果结果节点

- 空间详情明确显示“当前行星没有达到文明候选门槛”。
- 有生物圈但没有文明种子时，因果构建直接返回，没有建立负事实节点。
- 当前只为未形成生物圈建立负事实，没有覆盖未形成文明候选的分支。

可验证整改条件：

1. 为已有生物圈但未形成文明种子的行星建立稳定 `negative_fact` 节点。
2. 节点引用行星、生物圈、文明概率、生命等级、阈值规则和实际随机来源。
3. 空间详情可以直接打开该负事实节点。
4. 可见结果清单同时覆盖正事实与 UI 明确陈述的负事实。

## 5. P2 问题

### P2-01：六类随机操作可以整体同步改写

- 当前校验只能证明追踪内部的操作、候选和选中值彼此一致。
- 决定没有保存原始样本或可重放承诺，无法证明记录来自该 Seed、流和样本序号的真实执行。
- 把真实决定整体改写为内部合法的 `next`、`range`、`int`、`bool`、`pick` 或 `weighted` 记录并同步修改引用后，六种篡改均通过。

可验证整改条件：

1. 依据 Seed、算法版本、完整流身份和样本序号重放原始样本。
2. 按操作重新执行并核对真实选中值。
3. 候选与权重使用类型化规范载荷。
4. 增加六类操作整体转换及同操作参数同步修改负向测试。

### P2-02：文档测试数量和人工用例序号漂移

- 当前步骤 1 契约测试为 14 项，但退役登记仍记录 13 项。
- 架构文档仍使用“第 13 项”描述性能契约，而实际排列已经变化。

可验证整改条件：

1. 统一全部验证快照为当前真实数量。
2. 删除依赖排列顺序的“第 N 项”表述，改用稳定测试名称。
3. 优先使用测试输出或脚本生成数量快照。

## 6. 建议项

- 为随机决定建立版本化、类型化的结果绑定协议，保存领域作用目标、输出字段、集合身份和操作参数。
- 增加属性攻击与变异测试，自动执行引用搬运、操作互换、候选重写、清单整体重签和根证据同步修改。
