## 1. 整改结论

首轮评审登记的三项 P1 与三项 P2 均已完成实现、测试和文档整改，并在整改后原样通过完整 `npm run check:release`。

本报告只证明首轮问题已经具备关闭证据，不替代用户要求的全新 agent 独立复审。复审明确 P0、P1、P2 均为零前，步骤 1 仍不得标记完成。

## 2. P1 整改

### P1-01：观察投影伪造区域外来源

- `ObservationNode` 增加 `intensitySourceSubjects`，每项 `ObservationMeasure` 在同一份投影定义中同时生成数值、规则说明和实际来源主题。
- 文明、魔法和神性强度只引用当前区域实际参与计算的文明及必要神话操作数，魔法公式不再错误引用神话。
- UI 契约测试逐项重算行星的生命、文明、魔法、神性和因果强度，并证明区域外文明不会进入当前节点的直接原因。

关闭状态：已整改。

### P1-02：跨宇宙比较缺少双侧组合证据

- 新增 `ugs-law-comparison-evidence@1` 组合证据契约，绑定左右 Seed、生成身份、来源节点、领域评分、差值与最大差异稳定裁决。
- 左右因果图及其随机历史保持独立，单侧投影节点只陈述本侧评分和依据，不再声称单侧图可以独立证明完整差值。
- 新增身份错配、来源缺失、评分篡改、差值篡改和并列裁决负向测试。

关闭状态：已整改。

### P1-03：输入根无法证明来自本次生成

- 首轮整改时因果契约升级为 `ugs-causality@2`，新增 `CausalGenerationManifest`；第二轮复审整改进一步升级为 `ugs-causality@3`，补充类型化输入证据、集合边界和随机语义校验。
- 校验器拒绝缺失输入、额外输入、重排、输入类型错配、规范值错配和跨宇宙随机追踪。
- `RandomTraceSnapshot` 增加 `generationId`，低层因果图构建器退出公共 `src/sim` API，只接收受控生成信封。
- 运行时冻结覆盖生成清单与输入记录，防止校验后的根身份被修改。

关闭状态：已整改。

## 3. P2 整改

### P2-01：随机决策未验证所属随机流

- 随机决策索引保留所属流身份，校验器同时验证引用流、决策所属流、命名空间、样本序号、候选集和选中值。
- 新增两个流之间交换 `streamId` 或 `decisionId` 的负向测试，即使样本序号、候选集和选中值相同也必须拒绝。

关闭状态：已整改。

### P2-02：无效因果目标静默回退

- 明确区分“未指定目标”和“指定目标不存在”。
- 无效 `initialNodeId`、无效受控 `selectedNodeId` 和跨图节点 ID 均显示包含原 ID 的中文错误状态，不再选择其他节点。
- 独立组件与应用集成测试覆盖上述错误路径。

关闭状态：已整改。

### P2-03：文档门禁没有全部成为可执行测试

- `tests/architecture.test.ts` 已纳入全部步骤 1 因果核心、组件、UI 编排与样式文件的规模上限。
- 组件边界改为识别模拟层 `generate*` 与 `apply*` 能力，不再依赖少量旧函数名黑名单。
- 新增组件直接调用 `generateCausalUniverse` 的 AST 负向夹具，证明当前主生成入口不能越过应用模型边界。

关闭状态：已整改。
