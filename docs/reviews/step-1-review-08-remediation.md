## 1. 整改结论

第八轮复审提出的两项 P1 与两项 P2 已全部完成整改，完整发布门禁通过，步骤 1 等待下一名未参与实现、前八轮评审或整改的全新 agent 复审。

## 2. P1 整改

### P1-01：认证校验缓存污染

- 生产认证只在 `causality.ts` 私有 `WeakSet` 中登记，公共校验返回共享且冻结的成功结果。
- 成功结果对象和 `issues` 数组均不可修改，修改 `valid`、替换或追加 `issues` 均不能污染后续校验与序列化。
- 非认证图每次返回独立问题结果，修改问题对象不会影响下一次校验。

### P1-02：主要旧页面就地追因

- 概览指标、时间线事件、文明、神话、文明事件、干预结果、概率变化、指标变化和反噬事件均提供就地追因入口。
- 入口只携带领域主题，不要求用户搜索内部节点 ID；主题不存在或不能唯一解析时显示明确错误。
- 因果查询保存来源页面，用户可以返回原页面继续浏览。
- UI 与 Chromium、Firefox、WebKit、移动 Chromium 端到端测试覆盖完整进入和返回闭环。

## 3. P2 整改

### P2-01：统一定位器容器语义

- `CausalEntityLocator` 新增 `containerKind`，实体位置注册表统一登记实体类型、容器语义和可选主题后缀。
- 定位器创建、解析、领域值指纹和随机 `bindingKind` 使用同一注册信息。
- 删除实现与测试中的平行实体名单，字段和集合成员语义冲突会被拒绝。
- 因果契约升级为 `ugs-causality@8`，随机绑定契约升级为 `ugs-random-bindings@4`。

### P2-02：完整导出语法认证门禁

- 架构门禁覆盖匿名默认导出函数、默认导出变量与表达式、导出类、实例方法、静态方法和 `export assignment`。
- 新增对应失败夹具，真实模块仍只有 `buildUniverseCausalGraph` 能到达私有认证写入函数。
