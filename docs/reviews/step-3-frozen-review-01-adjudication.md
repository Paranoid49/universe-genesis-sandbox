## 3. P1-06 裁决依据喵

第二轮有限复查确认的剩余触发路径是：把合法信号及其证据替换为同一运行历史中的错误 `stateId`，再同步重算证据 ID、信号 ID 和外层校验和后仍能通过可信校验喵

第三轮整改已经从以下四层关闭该路径喵

1. `src/sim/observation-state.ts` 的 `observedStateId` 将被观察逻辑时刻映射到唯一可信状态身份：当前时刻使用当前 `state.id`，已完成时刻使用对应转换的 `afterStateId`，初始或尚无完成转换的历史时刻使用后继转换的 `beforeStateId` 喵
2. `src/sim/observation.ts` 在生成信号和证据时统一调用该映射，因此父信号与每条证据都绑定被观察时刻对应的同一精确状态身份喵
3. `src/sim/observation-verification.ts` 重新执行相同对象、方法和逻辑时刻的观察，并在可信语义比较中同时比较信号 `stateId` 与每条证据 `stateId`，因此“属于同一历史”不再能够替代“属于被观察时刻”喵
4. `src/sim/observation-identity.ts` 继续要求证据状态身份与父信号一致并校验内层身份，`src/sim/research-archive.ts` 则要求含信号记录必须经过可信校验器，外层校验和不能绕过可信重放喵

正式回归 `tests/step-3-observation.test.ts` 直接证明合法旧信号在宇宙继续推进后仍可恢复，同时把该信号替换为其他 `afterStateId` 或当前状态并同步重算全部身份后会以“结论或证据来源”拒绝喵

第三轮复查遗留的临时矩阵还覆盖逻辑时刻 0、1、2，每个时刻分别接受唯一对应状态，并拒绝另外两个同历史状态的全部同步重算替换；该矩阵复跑通过后已按资源约束删除喵

因此 P1-06 原 P1 判断在首次评审和前两轮复查时成立，但其当前剩余触发路径已不存在，裁定关闭喵

## 4. P2-01 裁决依据喵

当前事实文档没有继续把“状态身份只属于同一历史”误写成精确来源证明，而是与第三轮实现保持一致喵

- `docs/step-3.md` 明确写明可信校验器核对被观察时刻的精确状态身份，并声明同步重算内容哈希与外层校验和不能使同历史错误状态合法喵
- `docs/architecture.md` 明确区分内容哈希自洽与可信来源复验，并记录存档创建、写入、读取和页面恢复都必须核对精确状态身份喵
- `docs/quality-gates.md` 明确要求 `beforeStateId`、其他 `afterStateId` 和当前状态身份不得互换喵
- `docs/non-functional-requirements.md` 明确要求观察信号绑定被观察逻辑时刻对应的唯一状态，同历史错误状态不得进入研究记录喵
- `docs/legacy-module-migration.md` 与整改报告准确登记了同历史身份互换负向矩阵、合法旧信号继续推进后恢复和第三轮完整发布门禁证据喵

各当前事实文档在本裁决前统一标记为“第三轮整改与完整发布门禁已完成，等待第三轮有限复查”，没有提前宣称步骤 3 已完成喵
