## 2. P1-01：投影与生产事实认证分离

整改内容：

- `appendCausalProjections` 仍要求输入为生产认证事实图，但输出只作为冻结的非生产观察视图，不再写入生产认证闭包。
- 任意观察或解释投影即使结构合法，也会由公共校验返回 `UNTRUSTED_CAUSAL_GRAPH`。
- 非生产投影不能被公共序列化，也不能作为跨宇宙比较的可信事实来源。
- 因果查询界面仍可消费该只读视图，因为查询只依赖节点、边和双向邻接，不把视图提升为宇宙事实。
- 架构门禁检查 `certifyBuiltGraph` 的全部调用位置，确保只有受控宇宙生成工厂能够进入认证写入闭包。

测试证据：

- 普通观察投影、法则对比投影、重复 `share.code` 主题、任意标签和无关描述均不能通过公共认证或序列化。
- 生产基础图继续通过公共校验、序列化和比较。

## 3. P1-02：独立随机结果绑定转录

整改内容：

- 因果契约升级为 `ugs-causality@6`，随机结果绑定升级为 `ugs-random-bindings@2`。
- `RandomBindingTranscript` 在节点规格进入构建器时独立记录决定 ID、内部结果节点、结果主题、节点类型、绑定类型、原始作用域和领域输出路径。
- 最终节点完成稳定公开 ID 映射后，独立转录才映射节点 ID；公开绑定表则从最终节点引用另行生成。
- 受控工厂对独立期望转录与公开绑定表执行比较，不再克隆公开绑定表作为期望值。
- 输出路径改为稳定领域位置，包括指标、法则、时间线、空间层级、文明和分享字段，不再机械使用 `universe.${subjectId}`。
- 结构校验继续检查节点引用与公开绑定表内部一致；生产认证额外检查独立期望转录。

测试证据：

- 独立构建器测试先确认原始公开绑定与独立期望转录一致，再同步搬运节点引用、结果主题、节点类型和公开绑定后，独立期望转录比较明确失败。
- 生产图全部随机引用仍与公开绑定一一对应。

## 4. P2-01：补齐随机参数变异矩阵

新增 `range`、`int`、`bool`、`pick` 和 `weighted` 的同操作同步参数变异测试，分别覆盖范围、整数范围、概率、候选内容与顺序和权重变化。

每项测试同时证明变异记录可以保持结构内部自洽，但不能重新获得生产认证，从而区分结构校验与受控完整重放边界。
