## 2. P1 整改

### P1-01：随机引用可能挂到无关结果

- 每条随机引用新增 `resultSubjectId` 与原始 `scopeId`，构建器在节点加入时绑定真实结果主题。
- 校验器要求引用结果主题与当前节点一致、引用作用域与原决定一致，搬运引用或交换节点会返回 `INVALID_RANDOM_REFERENCE`。
- 时间线拆为基础数量边界和最终数量边界，基础事件引用基础边界，干预追加事件成为最终边界的直接原因。
- 新增引用搬运、集合边界与干预追加来源负向和正向测试。

关闭状态：已整改。

### P1-02：公共入口接受同步伪造图

- 增加进程内不可序列化的认证 WeakSet，受控生成在完整结构校验后认证并冻结图。
- 公共 `validateCausalGraph`、`assertCausalGraph` 和 `serializeCausalGraph` 只认可认证图，结构克隆会失去认证并返回 `UNTRUSTED_CAUSAL_GRAPH`。
- 结构一致性校验保留为内部受控入口，不从公共 `src/sim` API 导出。
- 投影要求输入图已认证，输出图使用认证来源的生成清单完成结构校验后获得新认证；跨宇宙比较同样拒绝未认证图。
- 新增同步伪造后对默认校验、序列化、投影和比较入口的负向测试。

关闭状态：已整改。

### P1-03：文明候选负事实不可追因

- 为已有生物圈但未形成文明种子的行星建立稳定 `negative_fact` 节点。
- 节点引用行星、生物圈、文明潜力、神性、时间线影响、文明生成公理和实际生物圈随机证据。
- 空间详情中的“未形成生命圈”和“未达到文明候选门槛”均提供“追溯未形成原因”按钮。
- 应用模型按主题定位唯一节点并打开因果页，UI 测试覆盖有效主题和未知主题。

关闭状态：已整改。

## 3. P2 整改

### P2-01：随机操作可以整体同步改写

- 随机追踪保存实际用于 PRNG 的完整 Seed 材料和每次抽样的 `sampleValue`。
- 校验器验证 Seed 材料与 Seed、规则版本、模板一致，并重新计算 Seed 指纹。
- 根据算法版本、Seed 材料、命名空间和样本序号重放原始样本。
- `next`、`range`、`int`、`bool`、`pick` 和 `weighted` 使用重放样本重新执行选择语义，核对范围、概率、候选、权重和选中值。
- 样本篡改、未知操作、伪造候选集和候选外结果均进入负向测试。

关闭状态：已整改。
