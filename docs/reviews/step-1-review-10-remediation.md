## 2. P1-01：字段、效果与观察结果细粒度追因

- 新增 `state_value` 因果投影类型和 `buildStateValueCausalProjection` 纯模拟层 API。
- 行星、生物圈、文明候选、文明属性与神话字段使用独立主题，点击时按需建立只读闭包投影，并直接引用对应实体、指标与生成公理。
- 字段投影不常驻复制进生产事实图，代表性旧宇宙的首次因果物化继续满足 1500 毫秒普通环境预算。
- `AttributeBar` 与其他字段展示使用 `data-result-subject` 标记实际可见结果，测试要求每个标记都包含同主题的生产追因或状态投影入口。
- 时间线每项事件效果连接既有 `event_effect` 节点，观察台当前事件连接生产事件节点，当前层级事件关联连接独立观察投影。
- 四类真实浏览器覆盖行星字段、文明字段、事件效果和观察事件关联的进入、原因、规则与返回闭环。

## 3. P1-02：聚合统计完整集合证据

- 空间统计拆为星系、恒星系、行星、生物圈和文明候选五个独立集合节点，总览节点只聚合这五项证据。
- 生物圈与文明候选统计同时引用每颗行星的正向成员或对应负事实，不再只引用星系集合。
- 文明统计拆为总数、路径、神话类型和高风险四个独立节点，神话统计直接引用神话节点，高风险描述明确记录阈值。
- 新增 `summaryCollectionEvidenceIssues` 与生产构建期 `assertSummaryCollectionEvidence`，校验统计描述、成员、负事实和筛选证据。
- 负向测试删除成员证据或伪造统计描述时必须失败。

## 4. P2-01：完整返回上下文

- 返回时无条件执行 `window.scrollTo(scrollX, scrollY)`，来源位置为零时也会恢复顶部。
- 因果投影请求携带与 `data-t` 无关的 `returnFocusKey`，观察投影、Seed 法则对比和状态值投影均使用唯一焦点令牌。
- 同一主题的观察操作区与时间线入口使用不同焦点键，返回时不会聚焦到同主题的另一个按钮。
- Chromium、Firefox、WebKit 和移动端 Chromium 均验证零坐标、非零坐标、重复主题与非主题投影入口。

## 5. P2-02：`export import` AST 门禁

- 架构分析器登记 `ImportEqualsDeclaration`，并把模块引用根标识符纳入符号引用图。
- 新增 `export import Alias = Unsafe` 与多级 `import Inner = Unsafe; export import Alias = Inner` 失败夹具。
- 真实认证模块仍只有 `buildUniverseCausalGraph` 能够到达私有认证写入函数。
