## 3. 正式问题

### P1-01 自主行动拒绝没有统一满足“稳定记录且不消费随机”

- 严重等级：P1。
- 冻结依据：C6；`2.4 意图与自主行动` 要求被约束、代价、权限或预算拒绝的行动保留稳定记录，且不产生随机消费或部分提交。
- 涉及位置：`src/sim/constitution-executor.ts:65-82`、`src/sim/autonomy.ts:86-88`。
- 触发方式一：为潮生自主规则增加随机效果，并让效果必然超过属性约束。第二步的两个行动均为 `constraint-rejected`，状态差异为零，但随机流 `sampleCount` 从 0 增长到 2，转换保存 2 条随机决定。
- 触发方式二：把 `maximumAutonomousActionsPerStep` 设为 1，同时保留两个会行动的主体。第二步直接抛出 `AUTONOMY_ACTION_BUDGET｜autonomy.actions`，原状态停留在第 1 步，历史中没有任何预算拒绝行动记录。
- 实际影响：拒绝动作会改变后续随机序列，或者完全丢失拒绝事实；相同后续输入可能因一次未提交动作而产生不同历史，玩家也无法追查预算拒绝原因。
- 判断依据：约束拒绝路径在效果采样后才校验约束；行动预算在准备阶段以异常中止，而不是形成正式拒绝执行记录。
- 整改建议：在消费随机前完成可判定的约束和预算预检；不能预检的候选应使用可回滚随机快照。把动作预算、权限、条件、代价、约束和裁决统一投影为稳定 `RuleExecutionRecord` 与 `AutonomousAction` 拒绝结果。
- 可验证关闭条件：分别增加条件、代价、权限、约束、裁决和预算拒绝测试；逐项断言状态对象、随机流、成功事件和差异不变，同时存在稳定错误码、字段路径、拒绝执行记录和行动记录。

### P1-02 多对象执行同一规则时，状态差异会连接到错误主体的执行记录

- 严重等级：P1。
- 冻结依据：C5；因果伪造威胁模型禁止把其他实体或无关执行记录写成行动原因。
- 涉及位置：`src/sim/runtime-causality-build.ts:157-164`。
- 触发方式：运行潮生世界两步。`rule.tide.signal@1` 分别为 `runtime.object.01` 和 `runtime.object.02` 生成两个执行记录。构建因果网络后，对象 02 的能量与信号差异均连接到对象 01 的执行记录。
- 实际影响：玩家查询对象 02 行动后果时，会得到对象 01 的条件、代价、约束和执行身份；双向因果链发生跨主体污染。
- 判断依据：差异归属只按 `ruleId` 使用 `find` 选择首个 applied 记录，没有同时匹配 `difference.objectId`。
- 整改建议：差异必须绑定稳定的执行记录 ID，或至少使用 `ruleId + objectId` 精确匹配；自主行动尝试还应拥有独立身份，避免同一对象上的多个策略共享执行结果。
- 可验证关闭条件：两个以上主体执行同一规则时，每项差异的执行记录、行动、实体和对象必须一致；加入反向断言，确认任何差异原因集中不包含其他主体的执行记录。

### P1-03 合法的小记忆容量会生成不可继续和不可保存的状态

- 严重等级：P1。
- 冻结依据：C7；`2.3 记忆、信念与认知变化` 要求有限容量淘汰后仍保持合法记忆来源和可恢复运行。
- 涉及位置：`src/sim/autonomy.ts:72-75`、`src/sim/constitution-autonomy-validation.ts:23-53`。
- 触发方式：把潮生策略 `memoryCapacity` 设为 1。该宪法能够通过校验；第二步为每个主体产生两条感知，但淘汰后只剩一条记忆，另一条信念的 `memoryIds` 为空。再次推进或创建运行存档均抛出 `AUTONOMY_BELIEF_MEMORY`。
- 实际影响：一个通过宪法校验的配置会先返回无效状态，随后在下一步或保存时失败，破坏可扩展规则包和运行可靠性。
- 判断依据：信念在记忆淘汰之后从 `nextMemories` 寻找来源；校验器没有约束容量与单步必要来源数量，也没有在返回新状态前执行完整性断言。
- 整改建议：信念应引用本步实际产生的来源记忆，再独立执行实体当前记忆窗口淘汰；同时明确“历史来源”和“当前可回忆记忆”的契约差异，并在转换提交前校验结果。
- 可验证关闭条件：容量为 1、感知字段多于 1、连续淘汰和保存恢复场景均能持续推进；每个信念仍引用同主体的真实历史记忆，当前记忆窗口不超过容量。

### P1-04 分支状态哈希与状态比较没有覆盖自主状态

- 严重等级：P1。
- 冻结依据：C10、C14；接口边界要求分支身份、状态哈希、历史哈希和比较覆盖自主行动与认知历史。
- 涉及位置：`src/sim/branch-validation.ts:13-15`、`src/sim/branch-comparison-evidence.ts:4-19`。
- 触发方式：克隆一个有效潮生状态，仅修改主体名称和记忆摘要。修改前后 `branchStateHash` 完全相同，`compareBranchObjects` 返回 0 项差异，而 `autonomy` 内容已经不同。
- 实际影响：状态哈希可以把自主认知不同的状态描述为相同；分支比较无法展示当前自主状态差异，并可能错误显示“状态已收敛”。迁移台账 S6-08 的完成证据因此不成立。
- 判断依据：`branchStateHash` 只包含 identity、clock、rules、objects 和 randomStreams；比较函数只展开世界对象。
- 整改建议：状态哈希纳入 topology 与 autonomy；比较契约增加实体、生命周期、记忆窗口、信念、意图、行动、关系和叙述差异，历史哈希继续保存实际转换路径。
- 可验证关闭条件：任意自主字段变化都会改变状态哈希；仅自主认知不同的共同祖先分支能在比较结果和页面中显示具体差异，同时相同当前认知但不同历史仍保持不同历史哈希。

### P1-05 行动因果页面可递归读取私有认知和内部 ID

- 严重等级：P1。
- 冻结依据：C11；`2.7 观察与界面` 明确禁止自动公开隐藏感知、私有记忆、完整信念和底层真相；界面泄漏威胁模型明确覆盖因果描述和内部类型 ID。
- 涉及位置：`src/components/pages/AutonomousEntitiesPage.tsx:31-34`、`src/sim/runtime-causality-build.ts:109-137`。
- 触发方式：在潮生实体页点击“查看行动因果”，随后按“为什么发生”依次进入意图、信念、记忆和感知。页面可显示 `拟执行 rule.tide.signal@1`、`相信 4，置信度 0.6`、`energy 被感知为 4`、`4｜确定性偏差`；行动和执行节点还显示 `applied` 等内部枚举。
- 实际影响：玩家无需取得任何观察授权即可读取完整私有认知链、内部字段 ID、规则 ID 和执行器枚举，直接破坏步骤 3 观察边界和步骤 6 认知隔离。
- 判断依据：页面直接消费完整 `RuntimeCausalNetwork`，递归按钮没有经过 `ObservationAccess`、可见性投影或脱敏层；当前架构测试只扫描页面源码字段名，未检查传入网络内容。
- 整改建议：为玩家建立受观察权限约束的行动因果投影，内部网络继续完整保存但不得直接交给页面。不可见认知使用“存在受限内部依据”等公开证据节点，公开规则、状态后果和行为信号使用中文稳定标签。
- 可验证关闭条件：递归遍历全部行动因果节点时，不出现私有感知值、记忆摘要、完整信念、置信度、内部对象／字段／规则 ID 或执行枚举；授权公开证据仍可双向查询并返回原实体上下文。

### P1-06 社会叙述与神话档案没有独立契约和身份

- 严重等级：P1。
- 冻结依据：C8；`2.5 实体关系、社会叙述与神话分层` 要求社会叙述、神话档案和真实实体使用不同契约与身份。
- 涉及位置：`src/sim/contracts/autonomy.ts:84-100`、`src/sim/contracts/autonomy-policy.ts:26-43`、`src/components/pages/AutonomousEntitiesPage.tsx:30`。
- 触发方式：潮生“丰潮传说”只被保存为 `AutonomousNarrative`；协议没有神话档案类型、身份或从叙述归档为神话的转换，页面将全部记录统一放在“公开叙述与神话”区域。
- 实际影响：系统无法区分普通共享叙述与神话档案，也无法对两者分别追因、比较和展示；文档所称的五层身份实际只实现了叙述与真实实体分离。
- 判断依据：当前公共自主协议仅有 `AutonomousNarrative`，没有独立 myth archive 契约或分类字段；同一集合承担两个冻结概念。
- 整改建议：增加独立版本化神话档案契约，明确其来源叙述、形成条件、身份和因果节点；页面分别展示实体叙述和神话档案，真实实体保持另一身份。
- 可验证关闭条件：同名叙述、神话档案和真实主体可同时存在且 ID 不同；神话只能从实际记忆、信念、交流或行动形成，不会自动创建真实主体或宇宙事实。

### P2-01 C12 的“360 像素纯键盘自主实体纵向闭环”没有被同一场景验证

- 严重等级：P2。
- 涉及位置：`tests/e2e/app.spec.ts:172-206`、`tests/e2e/app.spec.ts:240-333`。
- 触发或复现方式：潮生 E2E 覆盖移动 Chromium和主体观察，但使用直接点击且没有分支比较；360 像素纯键盘 E2E 使用奥术织网，没有自主实体入口。两段测试不能证明同一自主实体宇宙完成冻结要求的纯键盘观察、行动因果、分支、保存和恢复闭环。
- 实际影响：主体条件导航、实体选择和递归因果在窄屏键盘场景发生回归时，现有门禁可能仍保持绿色。
- 判断依据：C12 要求组合纵向闭环，当前证据是两个互不重合的场景。
- 整改建议：增加一条 360 像素潮生纯键盘 E2E，覆盖创世、两步形成、实体选择、行动因果、返回、实验分支、保存、刷新恢复和继续推进，并执行 axe 与横向溢出检查。
- 可验证关闭条件：该组合场景在冻结要求的真实浏览器范围内稳定通过，全部操作仅通过 Tab、方向键、Enter、Space 和文本键完成。
