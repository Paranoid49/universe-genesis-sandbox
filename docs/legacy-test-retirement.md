# 旧产品形状测试退役登记

状态：步骤 1 已完成分类登记和统一因果契约接管，实际退役仍须随步骤 2 至步骤 7 逐项执行；当前没有因步骤 1 提前删除后续兼容形状测试。

本文件是旧产品测试去留的唯一登记表。它不授权提前删除兼容测试，也不把旧产品形状提升为新产品永久需求。执行退役时，必须先提供新契约测试、迁移测试或明确拒绝测试，再删除旧断言。

## 1. 分类与关闭规则

| 分类 | 含义 | 当前门禁策略 |
| --- | --- | --- |
| 永久不变量 | 确定性、因果闭包、显式输入、错误可见性、只读投影、无障碍等长期要求 | 迁入新步骤契约测试后长期保留 |
| 旧基线兼容 | 为步骤 1 仍可运行的旧静态产品提供回归保护 | 在负责替换步骤完成前保留；不得解释为新产品形状 |
| 待退役形状 | 固定数量、固定领域、固定页面、固定对象层级、固定奇迹或固定存档格式 | 到登记步骤由新契约接管后删除 |
| 已由步骤 1 接管 | 原测试中的来源追踪、因果解释或 UI 可见投影追因职责已由统一因果契约覆盖 | 旧断言只保留兼容意义，权威验收转到步骤 1 契约、UI 投影与 E2E 纵向测试 |

关闭一条登记必须同时具备：新测试位置、旧断言删除或隔离的变更证据、兼容或拒绝策略，以及对应步骤文档更新。仅修改测试名称、降低数量或把入口隐藏起来不算退役。
