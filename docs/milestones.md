# 项目演进里程碑

本文件只维护长期路线、阶段状态和跨阶段边界，各阶段的详细需求与验收标准以对应的 `docs/phase-*.md` 为唯一事实来源。

## 1. 长期原则

- 确定性优先，同一显式输入必须产生同一宇宙。
- 自洽优先于真实，超自然能力同样必须具有规则、代价和边界。
- 先完成可验证体验，再增加模拟深度与视觉复杂度。
- 使用宇宙、空间、生命、文明、神话与干预的分层近似，不模拟无必要的微观实体。
- 每个阶段必须先有规格、再有实现、最后有自动化验收。

## 3. 后续阶段方向

### 阶段 7：可视化宇宙观察台

目标是把现有宇宙、星系、恒星系、行星、文明和时间线投影为只读可视化体验，同时保持文字路径、确定性和性能预算。

本阶段不得在独立规格完成前实现，准入要求见 [phase-7-readiness.md](phase-7-readiness.md)。

### 阶段 8：存档、分享与宇宙图书馆

候选范围包括本地或服务端存档、宇宙分支、收藏、搜索、版本迁移与公开宇宙展示，具体范围必须在阶段 7 稳定后重新评估。

### 阶段 9：创作者工具与内容扩展

候选范围包括规则包、内容包、模板编辑、校验工具和可控扩展接口，不预设插件运行时方案。

### 阶段 10：深度模拟与长期演进

候选范围包括更长时间尺度、文明交互、宇宙分支和可选择的高成本模拟，不承诺完整物理仿真。

## 4. 跨阶段质量要求

- 所有阶段必须通过 [quality-gates.md](quality-gates.md) 定义的完整门禁。
- 所有阶段必须遵守 [non-functional-requirements.md](non-functional-requirements.md) 的非功能要求。
- 生成结果变化必须升级规则版本并更新分享兼容策略。
- 阶段完成状态只能在规格、实现、测试和文档四者一致时更新。
