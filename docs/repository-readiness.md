## 4. 阶段 7 前置判断

阶段 7 只有在 [阶段 7 准入检查](phase-7-readiness.md) 和 `npm run check` 同时通过后才可以进入独立规格设计。后续开发需要保持以下约束：

- 不破坏 `seed + templateId + rulesetVersion + InterventionInput[]` 的确定性复现。
- 阶段 7 的可视化宇宙观察台必须先写独立规格。
- 阶段 7 不能反向静默改变阶段 0 至阶段 6 的既有结果。
- 可视化只能消费现有结构化数据或新增显式可复现输入。
- 新增测试必须继续覆盖阶段 1 至阶段 6 的既有验收项。

## 6. 建议的阶段 7 后续顺序

1. 先编写阶段 7 规格文档，明确可视化观察台只读消费哪些结构化数据。
2. 定义宇宙、星系、恒星系、行星和文明的可视化投影数据结构。
3. 明确时间播放如何消费已有时间线，不得改写生成结果。
4. 设计桌面与移动端可视化入口，避免破坏当前文字浏览路径。
5. 增加阶段 7 自动化验收测试，并继续验证 seed 与干预日志复现性。
