# 步骤 5 冻结验收口径：多本体宇宙沙盒

状态：已冻结。后续实现、完整评审、有限复查与裁决均不得新增要求或提高本文件的完成标准。

本文件以 [产品定义](product-definition.md)、[实施路线](milestones.md)、[旧产品模块迁移矩阵](legacy-module-migration.md)、[当前架构](architecture.md)、[质量门禁](quality-gates.md) 和 [非功能需求](non-functional-requirements.md) 为事实来源，冻结步骤 5 的必须实现内容、非目标、完成条件、兼容边界、质量要求和威胁模型。

## 1. 工作单元目标

步骤 5 必须把当前绑定固定模板、固定对象、固定规则、固定指标、固定观察方式和固定干预字段的运行产品，替换为由可组合宇宙宪法驱动的统一多本体运行时。

玩家必须能够选择完整宪法预设，也能够在兼容约束内组合宪法模块，创建运行基础显著不同的宇宙。所有通过校验的宇宙仍使用同一套状态转换、确定性随机、因果追踪、观察、实验、分支、比较、保存和恢复基础设施。

步骤 5 的成功不以内容数量、惊奇程度、生命出现率或叙事效果衡量，只以规则自洽、统一执行、动态能力、因果闭包和完整可运行闭环衡量。

## 2. 必须实现的要求

### 2.1 宇宙宪法与规则包

- 新增独立版本的宇宙宪法、规则模块、规则包和组合结果契约。
- 宇宙宪法必须至少包含本体、作用、约束、优先级、时间、拓扑、认知边界、可观察量、事件分类、干预能力和宇宙边界十一类模块引用。
- 每个模块必须具有稳定 ID、独立版本、模块类别、依赖、冲突、公开说明和规范内容指纹。
- 宇宙定义身份必须绑定 Seed、宪法版本、全部模块 ID 与版本、规范模块内容、初始输入和规则执行器版本。
- 模块组合必须生成稳定且可序列化的宪法快照，运行时不得依赖可变全局注册表恢复既有宇宙语义。
- 规则包只能使用受控声明式条件、效果、约束、代价和裁决操作，不允许规则包携带或执行任意 JavaScript、动态代码、网络请求、浏览器 API 或系统时间读取。
- 新增合规规则包时，不得修改规则执行器或既有规则包内部实现；参考规则包不得通过产品代码中的规则包 ID 分支获得专用流程。

### 2.2 统一规则执行

- 所有规则包必须通过同一个状态转换入口执行，不允许为物理、魔法、梦境或其他本体建立互不兼容的模拟主流程。
- 条件判断、约束检查、效果应用、代价结算、冲突裁决和确定性选择必须形成显式、稳定且可重放的执行记录。
- 同一转换中多个规则写入同一目标时，必须由宪法声明的优先级和裁决策略决定结果；缺少裁决或存在无法消解的平级冲突时必须拒绝宪法或拒绝转换，不能依赖数组顺序、对象键顺序或运行平台行为。
- 代价和效果必须原子提交；代价不足、约束失败或效果无效时不得留下部分状态修改。
- 每个实际状态差异必须引用实际执行规则、满足的条件、通过的约束、裁决结果、代价记录和适用的随机决定。
- 确定性随机只能在规则允许的候选范围内选择，不能替代规则、条件或前置状态成为语义原因。
- 规则执行必须设置明确预算，阻止无限规则链、无限效果展开、无界递归和单步资源耗尽。

### 2.3 动态世界结构

- 运行对象的种类、属性定义、关系和拓扑必须由宇宙宪法决定，不得要求固定“星系—恒星系—行星”层级。
- 通用状态必须支持图关系或等价的非传统拓扑；没有传统空间的宇宙仍能创建对象、演化、观察、实验、分支和保存。
- 逻辑时钟继续作为确定性排序基础，时间模块可以定义线性、循环或分段显示与规则节律，但不能读取墙上时间或改变已提交输入顺序。
- 指标只能是状态或观察结果的派生投影，指标 ID、名称、计算来源和可见性由宪法定义，不能成为独立事实来源。
- 事件只能从已提交状态差异和宪法事件分类规则产生；不存在预制未来，事件类别不能固定要求生命、文明、神话或终局。
- 认知模块在本步骤只定义可见性、知识边界和步骤 6 自主实体的兼容接口，不实现自主实体感知、记忆、信念、意图或行动。

### 2.4 动态观察、实验与干预

- 观察方式 ID、名称、尺度、适用对象、可见字段、证据生成和无结果语义必须由宪法提供，不能继续使用固定四种方式作为公共类型上限。
- 观察仍必须消费可信历史和有限可见信号，不能因规则包动态化而绕过步骤 3 的认知边界读取完整底层状态。
- 宪法可以不提供某类观察方式；界面必须隐藏不适用入口，并对“当前没有可用方式”给出可读说明。
- 界外实验与宇宙内干预继续保持步骤 4 的身份、权限和历史语义分离。
- 实验字段、干预能力、目标类型、参数、约束、代价和拒绝原因由宪法声明，不再固定为 `cohesion`、`energy`、奇迹点或强制反噬。
- 不提供宇宙内干预能力的宪法不得显示空白干预表单；玩家仍可建立界外实验分支。
- 接受和拒绝的输入结果必须保留稳定错误码、字段路径、适用规则和前置状态，拒绝不能伪造状态转换或因果节点。

### 2.5 创世与界面

- 默认新产品创世入口必须从固定十模板选择器替换为宪法预设选择和兼容模块组合。
- 创世界面必须在启动前显示宪法摘要、公开公理、时间模型、拓扑、观察能力、干预能力和校验结果。
- 模块组合非法时必须在页面内给出稳定错误码、字段路径和可读原因，不能启动部分有效的宇宙。
- 当前宇宙、观察、实验、历史分支、历史和存档页面必须消费动态契约，不得通过规则包 ID、传统对象类型或旧领域枚举决定固定页面结构。
- 标签层必须为动态模块、对象、指标、观察方式、事件和干预能力提供规则包声明的中文显示名称；未知类型不能泄露内部枚举或静默回退为错误含义。
- 360 像素窄屏和纯键盘必须能够完成宪法选择或组合、校验、创建、推进、观察、干预或明确无干预、分支、保存和恢复闭环。

### 2.6 参考规则包

步骤 5 固定提供以下三个参考预设，用于证明统一引擎没有绑定单一世界观。

1. `material-expanse@1`：线性时间、层级空间拓扑、物质结构与能量变化，允许结构观察和有限状态干预。
2. `arcane-weave@1`：循环时间、关系网络拓扑、法力与共鸣作用，提供不同于物质宇宙的观察方式、事件分类和干预能力。
3. `dream-flux@1`：分段时间、非空间语义关系拓扑、连贯性与可塑性变化，不包含传统星系、恒星、行星、生命或文明专属指标。

三个预设必须满足以下约束：

- 都通过同一个规则执行器、状态转换入口、因果网络、观察访问端口、分支系统和存档系统运行。
- 至少两种时间模型、三种拓扑或对象模式、三组指标、三组事件分类和两组不同干预能力能够在产品界面中被实际观察。
- `dream-flux@1` 必须完成创建、连续演化、观察、界外实验、分支比较、保存和恢复，并且页面不出现无意义的星系、生命、文明、神话或奇迹占位内容。
- 至少提供一个不属于三个完整预设的合法模块组合测试夹具，证明组合能力不是预设切换的别名。

### 2.7 因果、分支与保存

- 宇宙宪法及其完整模块快照必须成为运行因果网络的合法根因。
- 规则条件、约束、裁决、代价、效果、随机决定、状态差异、转换和事件必须能够从结果反向追溯，也能从原因查看已经产生的后果。
- 完备原因完全相同时，三个参考规则包分别产生完全相同的状态、随机决定、执行记录、因果网络和观察结果。
- 分支身份、状态哈希和历史哈希必须覆盖宪法身份及规则执行记录；不同宪法的分支不能冒充共同祖先分支。
- 同一宪法下回到过去修改必须继续创建新分支，父分支保持不可变。
- 存档必须保存完整宪法快照、状态、随机流、输入、转换、规则执行记录和因果重建所需数据，恢复后的下一步必须与未中断运行一致。
- 创世条件包和历史分支包必须携带宪法身份与模块快照；缺失模块、版本错配、内容指纹错配或重签篡改必须明确拒绝。

## 3. 明确不实现的内容

- 不实现步骤 6 的自主实体、生命形成机制、实体感知、记忆、信念、意图、行动、社会关系或自主历史。
- 不实现步骤 7 的长期跨版本迁移体系、静态创作档案、结构化编年史和正式创作文件导出。
- 不实现联网规则包市场、远程下载、第三方代码插件、用户脚本或任意代码执行。
- 不实现完整可视化规则编辑器；本步骤只提供受控模块选择、组合、校验和摘要。
- 不实现分支合并、多人协作、云同步、服务器模拟或后台持续运行。
- 不要求精确复刻现实物理，也不提供生命、文明、神明、魔法或叙事高潮配额。
- 不增加任务、成就、胜利条件、探索评分、重要性评分或产品预设目标。
- 不修改步骤 1 旧静态 UGS070、I1 和 A1 的隔离兼容语义。

## 4. 接口、数据与兼容边界

- 步骤 1 至步骤 4 已完成的因果闭包、状态唯一事实来源、逻辑时钟、可恢复随机流、认知边界、不可变分支、分享类型和原子存储不变量必须保持。
- 宇宙定义、状态、转换、观察、分支、存档和分享协议如因新字段升级版本，必须同步升级其身份、校验器、错误码、测试和当前文档，不能让不同语义共用旧版本号。
- 步骤 4 的运行分支、检查点和历史包不得按新的宪法执行器重新解释后声称历史等价。
- 步骤 4 数据可以明确拒绝、进入隔离只读查看，或转换为具有新宇宙定义和新分支身份的宪法预设；转换界面必须声明历史不等价。
- 旧十种 `templateId` 只能用于旧静态数据读取和显式的宪法预设映射，不能进入新规则执行器的公共接口或限制对象、指标、事件和观察结构。
- 固定六领域、七指标、固定纪元、三级空间、固定观察叠层和固定奇迹只能存在于旧版隔离兼容路径。
- 研究记录只能在宪法身份、历史身份、状态身份、观察方式和可信重算全部匹配时恢复，不能把旧固定观察信号自动包装为新动态信号。
- 新运行数据继续使用版本化 IndexedDB 适配器，完整状态、宪法、历史和因果数据不得写入 `localStorage`。
- 新产品公共导出必须提供宪法、模块、动态规则、动态观察、动态干预和存档契约；旧固定领域类型只能从隔离兼容入口导出。

## 5. 可验证完成条件

- C1：玩家能够选择三个参考宪法预设，并能完成至少一个跨预设模块组合的校验与创建。
- C2：三个参考宇宙通过同一模拟入口创建和推进，生产执行器不存在按规则包 ID 分流的专用逻辑。
- C3：相同 Seed、宪法快照、状态、随机流和有序输入产生完全相同的状态、执行记录、事件、因果网络和观察结果。
- C4：规则冲突按照宪法稳定裁决并记录证据；缺少裁决、依赖缺失、依赖循环、冲突模块和无效效果被明确拒绝。
- C5：规则代价和效果原子提交，失败不会留下部分状态、随机流、输入日志或转换记录。
- C6：物质、魔法和梦境参考宇宙均能形成至少一个由真实状态差异产生的事件，并完成双向因果查询。
- C7：`dream-flux@1` 在没有传统空间、生命和文明指标的情况下完成演化、观察、实验、分支、保存与恢复。
- C8：观察方式、指标、事件分类和干预能力随宪法变化，界面没有固定旧领域的必选入口或空白占位页面。
- C9：没有宇宙内干预能力的组合显示明确说明，提供能力的宪法能够提交接受或拒绝结果并正确进入因果历史。
- C10：三个参考宇宙均能从历史时刻创建分支、比较共同祖先后的状态和因果差异，并保持父分支不变。
- C11：三个参考宇宙保存、刷新、恢复后的下一步与未中断运行完全一致。
- C12：宪法、存档和分享内容发生模块缺失、版本错配、指纹错配、优先级篡改或执行记录篡改时被稳定拒绝。
- C13：旧模板、固定法则、固定指标、固定空间树、固定观察叠层和固定奇迹不能从新主流程依赖图到达。
- C14：桌面和 360 像素窄屏均能以键盘完成宪法创建、运行、观察、实验或干预、分支、保存和恢复，并通过 axe 检查。
- C15：迁移矩阵中归属于步骤 5 的模板、法则、指标、空间、观察、选择、标签、干预、内容和公式条目均具备实现、测试、兼容和文档证据。
- C16：完整发布门禁、步骤 5 专属性能门禁和全新独立 Agent 首次完整评审完成，不存在有效未关闭的 P0 或 P1，P2 均获得明确处置。

## 6. 质量要求

- 必须通过完整 `npm run check:release`，不得降低现有全局覆盖率、文件覆盖率、构建、体积、性能、浏览器或无障碍门禁。
- 宇宙宪法校验、规则执行、规则执行记录、动态观察和动态输入的关键新文件必须设置独立覆盖率阈值，最低为语句 90%、分支 82%、函数 90%、行 90%。
- 新创世与宪法组合页面适用步骤 3、步骤 4 页面门禁，最低为语句 80%、分支 75%、函数 80%、行 80%。
- 步骤 5 独立非覆盖率性能门禁必须执行两次预热和至少十二个样本，覆盖三个参考预设与合法组合夹具。
- 宪法规范化与完整校验 P90 小于 100 毫秒，初始状态创建 P90 小于 500 毫秒，一百步演化 P90 小于 2500 毫秒，动态观察 P90 小于 100 毫秒，存档恢复与完整性校验 P90 小于 1500 毫秒。
- 性能优化不得改变宪法身份、模块顺序、规则裁决、代价、状态序列、随机决定、执行记录、因果节点、观察结果或存档内容。
- 创世选择、模块组合、校验错误、观察方式切换和干预参数输入的页面反馈在普通桌面环境中 100 毫秒内可见。
- 生产构建继续遵守 JavaScript gzip 总体积 115 KiB、日常保护线 110 KiB和 CSS gzip 总体积 10 KiB、日常保护线 8 KiB；如确需改变，只能通过独立架构决策记录，不得在实现或评审中静默放宽。
- 新增文件继续遵守现有文件规模、职责边界、循环依赖和公共入口门禁；规则执行器、规则包目录、UI 协调层和存储层必须分离。

## 7. 必要威胁模型

- 宪法身份伪造：修改模块内容后重算外层校验和，仍试图冒充原宇宙。
- 模块替换攻击：保留模块 ID 和版本但替换规范内容，或使用相同内容冒充不同版本。
- 依赖与冲突攻击：缺失依赖、循环依赖、冲突模块共存或非法类别重复。
- 裁决不确定性：平级规则依赖数组顺序、对象键顺序、浏览器或平台差异产生不同结果。
- 部分提交：代价已经扣除但效果失败，或部分效果写入后约束才失败。
- 执行记录篡改：状态结果保持不变但条件、约束、裁决、代价、规则或随机来源被替换。
- 资源耗尽：规则链、效果展开、对象创建、关系扩展或因果路径在单步中无界增长。
- 越权观察：动态观察定义读取未授权字段、其他宇宙状态或当前历史之外的数据。
- 越权干预：规则包声明之外的目标、字段、参数或能力进入状态转换。
- 跨宪法分支伪造：不同宪法或不同模块内容的历史被声明为具有共同祖先。
- 旧数据冒充：步骤 4 的模板分支、UGS070、I1 或 A1 被自动包装成等价的新宪法历史。
- 规则包专用后门：生产执行器或界面通过参考规则包 ID 硬编码结果、页面或能力。
- 动态代码执行：规则包字段触发脚本、原型污染、网络访问、浏览器 API、系统时间或非有限数值。
- 认知边界泄漏：非传统宇宙通过通用页面、标签回退或旧选择器泄露完整底层事实。

## 8. 评审约束

首次完整评审 Agent 只能依据本文件评审步骤 5 及其对步骤 1 至步骤 4 的直接回归，不得评价步骤 6、步骤 7 或增加未来能力要求。

评分只说明现状，不构成额外完成门槛。所有 P0 和 P1 必须引用本文件的明确条款或步骤 1 至步骤 4 必须保持的核心不变量。

每轮整改使用全新独立 Agent，有限复查只检查未关闭问题、本轮修改及其直接回归。最多三轮有限复查，仍有争议时启动全新裁决 Agent。
