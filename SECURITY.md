# 安全策略

## 支持状态

本项目尚处于最小可用版本之前，还没有正式发布版本。当前安全检查重点是仓库卫生、依赖使用、构建工具，以及未来客户端数据处理方式。

## 报告方式

如果你发现安全问题，请不要在公开 issue 中写出利用细节。

优先使用 GitHub 私密报告，或通过私密渠道联系仓库维护者。如果暂时没有私密报告入口，可以创建一个最小公开 issue，说明“有安全报告需要处理”，但不要包含技术细节或复现步骤。

## 当前安全约束

- 不要提交密钥、令牌、私钥或本地 `.env` 文件。
- 不要在未说明用途的情况下加入遥测、远程执行或网络请求。
- 除非后续阶段明确引入后端服务，否则生成逻辑保持确定性和本地优先。
- 创建应用骨架后，应提交依赖锁定文件。
